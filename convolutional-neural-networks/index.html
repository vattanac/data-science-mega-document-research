1. Why CNNs?

Standard neural networks (MLPs) treat every pixel as an independent input. A 256×256 RGB image has 196,608 inputs — connecting all of them to even a small hidden layer creates millions of parameters, most of which are redundant.

CNNs exploit three key properties of visual data:

  • Local connectivity: a pixel is most related to its neighbors, not pixels far away.
  • Parameter sharing: the same pattern (edge, texture) can appear anywhere in the image.
  • Translation invariance: a cat is a cat whether it's in the top-left or bottom-right.
Instead of looking at every pixel simultaneously (like an MLP), a CNN slides a small magnifying glass (filter) across the image, looking for specific patterns. The same magnifying glass is used everywhere — so one filter can detect horizontal edges wherever they occur.

2. The Convolution Operation

A convolution slides a small filter (kernel) over the input, computing element-wise multiplication and sum at each position to produce a feature map.

(I * K)[i,j] = ΣₘΣₙ I[i+m, j+n] · K[m,n]

A 3×3 filter has only 9 learnable parameters, but it gets applied at every position in the image. This is parameter sharing — dramatically fewer parameters than a fully connected layer.

Example: 3×3 Edge Detection Filter

A vertical edge detector might look like:

K = [[-1, 0, 1], [-1, 0, 1], [-1, 0, 1]]

This filter produces high values where there's a sharp horizontal brightness change (vertical edge). The network learns these filters from data — you don't design them manually.

Multiple Filters = Multiple Feature Maps

Each convolutional layer applies multiple filters (e.g., 32, 64, 128). Each filter detects a different feature. The output is a stack of feature maps — one per filter. This is the "depth" dimension of the output.

Input: H×W×C_in → Conv(F filters, k×k) → Output: H'×W'×F

3. Padding & Stride

Padding

Without padding, the output shrinks: a 5×5 input with 3×3 filter produces 3×3 output. Padding adds zeros around the border to control output size.

  • Valid (no padding): output = (n - k + 1). Shrinks each layer.
  • Same (zero padding): pad with p = (k-1)/2. Output = same size as input.

Stride

Stride controls how many pixels the filter jumps. Stride=1 slides one pixel at a time. Stride=2 skips every other position, halving the output size.

Output size = ⌊(n + 2p - k) / s⌋ + 1

Where n=input size, k=kernel size, p=padding, s=stride.

Common pattern: 3×3 filters with same padding and stride 1 for most layers. Use stride 2 (or pooling) to downsample.

4. Pooling Layers

Pooling reduces spatial dimensions, making the network more efficient and providing some translation invariance.

  • Max Pooling (most common): take the maximum value in each window. Preserves the strongest activations.
  • Average Pooling: take the average in each window. Smoother but loses sharp features.
  • Global Average Pooling: average the entire feature map to a single value. Often used before the final classifier layer.

Typical: 2×2 max pooling with stride 2 → halves spatial dimensions. A 28×28 feature map becomes 14×14.

Max pooling is like summarizing a paragraph with just the most important sentence. You lose detail but keep the key information, and the summary is shorter (smaller spatial dimensions).

5. CNN Building Blocks

Typical CNN Architecture

  1. Input: raw image (e.g., 224×224×3 for RGB).
  2. Conv + ReLU + Pool block (repeated): extract increasingly abstract features while reducing spatial size.
  3. Flatten: reshape 3D feature maps into 1D vector.
  4. Fully Connected layers: classify based on extracted features.
  5. Output: softmax for classification (or linear for regression).

Batch Normalization

Normalize activations within each mini-batch. Placed after conv (before or after ReLU). Stabilizes training, allows higher learning rates, acts as mild regularization.

1×1 Convolutions

A 1×1 filter doesn't look at spatial context — it combines information across channels. Used to reduce/increase channel depth cheaply. Key building block in modern architectures (Inception, ResNet bottlenecks).

6. Feature Hierarchy

The magic of deep CNNs: each layer builds on the previous one to detect increasingly complex patterns.

Layer DepthWhat It DetectsExamples
Layer 1Low-level featuresEdges, corners, color gradients
Layers 2–3Mid-level featuresTextures, patterns, simple shapes
Layers 4–5High-level featuresObject parts (eyes, wheels, windows)
Final layersSemantic featuresWhole objects, scenes, concepts
Early layers are universal — edges and textures appear in all image types. Later layers become task-specific. This is why transfer learning works: reuse early layers, retrain later ones.

ResNet's Skip Connections

output = F(x) + x (residual: learn the difference, not the full mapping)

Skip connections allow gradients to flow directly through the network, enabling training of very deep networks (100+ layers) without vanishing gradients. This was a breakthrough that enabled modern deep learning.

8. Transfer Learning

Training a CNN from scratch requires millions of images. Transfer learning reuses a network pre-trained on a large dataset (ImageNet: 1.2M images, 1000 classes).

Two Approaches

  1. Feature extraction: freeze all pre-trained layers, replace only the final classifier. Train only the new head. Fast, works with small datasets.
  2. Fine-tuning: unfreeze some later layers and retrain with a very small learning rate. More powerful, needs more data.
Transfer learning is the most important practical technique in deep learning. With it, you can get state-of-the-art results on custom tasks with just hundreds of images instead of millions.

9. Practical Tips

  • Start with transfer learning (ResNet-50, EfficientNet-B0) unless you have millions of images.
  • Data augmentation is essential: random flips, rotations, crops, color jitter. More effective than collecting more data.
  • Batch normalization after every conv layer for stable training.
  • Use 3×3 filters — two 3×3 layers have the same receptive field as one 5×5, but fewer parameters and more nonlinearity.
  • Double channels when halving spatial size (e.g., 64 channels at 56×56 → 128 channels at 28×28).
  • Global average pooling instead of flatten+FC to reduce parameters at the end.
  • Learning rate: start with 1e-3 for training from scratch, 1e-4 for fine-tuning.

13. Summary & Key Takeaways

  1. Convolution exploits locality and parameter sharing — slides small filters across images to detect patterns.
  2. Pooling reduces spatial dimensions and provides translation invariance.
  3. Deeper layers detect increasingly abstract features: edges → textures → parts → objects.
  4. ResNet's skip connections solved vanishing gradients, enabling 100+ layer networks.
  5. Transfer learning is the practical default — start with pre-trained models, fine-tune on your data.
  6. Data augmentation is essential for preventing overfitting on small datasets.
  7. Use 3×3 filters, batch normalization, and global average pooling as standard building blocks.
  8. Modern CNNs are the backbone of computer vision: image classification, object detection, segmentation, and more.
