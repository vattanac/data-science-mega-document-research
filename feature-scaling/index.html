Feature Scaling

Normalization vs Standardization

Why Feature Scaling?

The Core Problem

Machine learning algorithms often assume features are on a similar scale. Without feature scaling, algorithms can behave unexpectedly, converge slowly, or fail entirely. Here are the main reasons why feature scaling matters:

Gradient Descent Convergence

When features have different scales, the loss surface becomes elongated (elliptical contours). Gradient descent takes small steps and zigzags instead of converging directly to the minimum. Scaling features produces circular contours, allowing gradient descent to converge efficiently and in fewer iterations.

Distance-Based Algorithms

Algorithms like K-Nearest Neighbors (KNN), K-Means, and SVM rely on distance metrics (Euclidean distance, Manhattan distance, etc.). Features with larger scales dominate the distance calculation, making smaller-scale features irrelevant. Scaling ensures all features contribute equally to distance computations.

Feature Dominance

Without scaling, a feature with values ranging from 1,000 to 10,000 will dominate over a feature with values ranging from 0 to 1. This biases the model toward the larger-scaled feature, regardless of its actual predictive power.

Key Insight: Feature scaling doesn't change the underlying relationships in your data—it just ensures all features are treated fairly by your algorithm.

Min-Max Normalization

Also called: Feature Scaling or Rescaling

Min-Max normalization scales features to a fixed range, typically [0, 1]. This is the most intuitive scaling method and is widely used in practice.

Formula

x_scaled = (x - min) / (max - min)

Example

Dataset: [10, 20, 30, 40, 50]

  • min = 10, max = 50
  • 10 → (10-10)/(50-10) = 0/40 = 0.0
  • 20 → (20-10)/(50-10) = 10/40 = 0.25
  • 30 → (30-10)/(50-10) = 20/40 = 0.5
  • 40 → (40-10)/(50-10) = 30/40 = 0.75
  • 50 → (50-10)/(50-10) = 40/40 = 1.0

Advantages

  • Intuitive: All values fall in [0, 1], easy to interpret.
  • Preserves relationships: Relative distances between points are preserved.
  • Works for bounded data: Great when you know min/max values.
  • Fast: Simple linear transformation.

Disadvantages

  • Sensitive to outliers: A single outlier can compress most data into a small range.
  • Requires min/max: Must recompute on new data; can cause extrapolation beyond [0,1].
  • Assumes bounded data: Not ideal for unbounded distributions.
Use case: Neural networks, image processing (pixel values), and when you need values in a specific bounded range.

Standardization (Z-Score Normalization)

Also called: Standardization or Z-score normalization

Standardization transforms features to have mean = 0 and standard deviation = 1. This is the most common scaling method in machine learning and statistics.

Formula

x_scaled = (x - μ) / σ

where μ is the mean and σ is the standard deviation

Example

Dataset: [10, 20, 30, 40, 50]

  • μ (mean) = 30, σ (std) ≈ 14.14
  • 10 → (10-30)/14.14 ≈ -1.41
  • 20 → (20-30)/14.14 ≈ -0.71
  • 30 → (30-30)/14.14 = 0.0
  • 40 → (40-30)/14.14 ≈ 0.71
  • 50 → (50-30)/14.14 ≈ 1.41

Advantages

  • Works well with linear models: Recommended for linear regression, logistic regression, and SVM.
  • Works for unbounded data: No assumptions about min/max; handles infinite ranges.
  • Statistically sound: Follows normal distribution assumptions in many algorithms.
  • Less sensitive to outliers: Better than min-max when outliers are present (though still affected).

Disadvantages

  • Less intuitive: Values can be negative and > 1; harder to interpret directly.
  • Slightly sensitive to outliers: Outliers affect both mean and standard deviation.
  • Assumes normal distribution: Best for normally distributed features.
Use case: Linear models (regression, logistic), distance-based algorithms (KNN, K-Means, SVM), and whenever you expect roughly normally distributed data.

Robust Scaling

For outlier-heavy datasets

When your data contains significant outliers, both min-max and standardization can be distorted. Robust scaling uses the median and interquartile range (IQR), which are robust to outliers.

Formula

x_scaled = (x - median) / IQR

where IQR = Q3 - Q1

How It Works

  • Median: Middle value of the dataset (50th percentile). Much less affected by outliers than the mean.
  • IQR: Range between 25th percentile (Q1) and 75th percentile (Q3). Contains the middle 50% of data; ignores extreme values.

When to Use

  • Dataset contains known outliers
  • You want a more conservative scaling method
  • Distribution is skewed or non-normal
  • Using algorithms sensitive to scale but robust to outliers (e.g., tree-based models with outliers)
Caveat: Robust scaling doesn't eliminate outliers—it just scales them more reasonably. Consider outlier removal or treatment as a separate preprocessing step if needed.

When NOT to Scale

Feature scaling is beneficial for most algorithms, but some models are completely invariant to feature scaling:

Decision Trees & Random Forests

Decision trees split on feature values, not distances. A feature with values [0, 1] vs [0, 1000] will produce the same splits and tree structure. Scaling has no effect on tree-based models.

Gradient Boosting Models

XGBoost, LightGBM, and CatBoost are tree-based and scale-invariant. They find optimal splits regardless of feature scale.

Rule-Based Models

Models that work with categorical rules or thresholds (rule-based systems, neural networks for classification on unscaled data) may not benefit from scaling.

When You Might Still Scale Tree Models

  • For consistency across your preprocessing pipeline
  • If using hyperparameter tuning that assumes scaled features
  • When combining tree models with distance-based models in an ensemble
Rule of thumb: If your algorithm uses distances, gradients, or assumes normally distributed features → scale. If it uses thresholds or splits → optional.

Quick Decision Guide:
  • 🔹 Linear/Distance-based (SVM, KNN, K-Means, Linear Regression) → Use Standardization

  • 🔹 Tree-based (Decision Trees, Random Forest, XGBoost) → Skip scaling
