1. What Is Regression?

Regression predicts a continuous output (number) from input features. Unlike classification (which predicts categories), regression answers "how much?" or "how many?"

Regression is like fitting a trend line through a cloud of points. You're looking for the underlying pattern — the signal — while ignoring the noise. The better your line captures the true relationship, the better your predictions.

Examples: predicting house prices from square footage, estimating salary from years of experience, forecasting temperature from historical data.

2. Simple Linear Regression

The simplest model: one input feature, one output, and a straight line.

ŷ = w₁x + b (or equivalently: ŷ = β₀ + β₁x)

The goal: find w (slope) and b (intercept) that minimize the sum of squared errors between predictions and actual values.

Ordinary Least Squares (OLS)

min Σᵢ (yᵢ − ŷᵢ)² = min Σᵢ (yᵢ − w₁xᵢ − b)²

The closed-form solution:

w₁ = Σ(xᵢ − x̄)(yᵢ − ȳ) / Σ(xᵢ − x̄)² b = ȳ − w₁x̄

w₁ is the covariance of x,y divided by the variance of x. b ensures the line passes through the centroid (x̄, ȳ).

Interpretation

  • Slope (w₁): for every 1-unit increase in x, ŷ changes by w₁ units.
  • Intercept (b): predicted value when x = 0 (may not be meaningful if 0 is outside the data range).
  • R² (coefficient of determination): fraction of variance in y explained by the model. R²=1 is perfect, R²=0 means no better than predicting the mean.

3. Multiple Linear Regression

Extend to multiple features: predict y from x₁, x₂, ..., xₚ.

ŷ = w₁x₁ + w₂x₂ + ... + wₚxₚ + b = Xw + b

In matrix form, the OLS solution is:

w = (XᵀX)⁻¹ Xᵀy

Each coefficient wⱼ tells you the effect of feature j on y, holding all other features constant. This is the key advantage of multiple regression — it isolates individual effects.

Multicollinearity warning: If features are highly correlated, (XᵀX) becomes nearly singular, coefficients become unstable and hard to interpret. Check using VIF (Variance Inflation Factor) > 5–10 signals a problem.

4. Polynomial Regression

Capture nonlinear relationships by adding polynomial terms:

ŷ = w₁x + w₂x² + w₃x³ + ... + wₙxⁿ + b

This is still linear regression — it's linear in the parameters (w), just nonlinear in x. You're simply adding engineered features (x², x³, ...).

  • Degree 1: straight line (underfitting if data is curved).
  • Degree 2–3: captures curves, often good enough.
  • Degree 10+: wiggly, overfits, follows noise instead of signal.
Polynomial regression is like choosing how flexible your ruler is. A rigid ruler (degree 1) gives a straight line. A bendy ruler (degree 3) follows curves. A spaghetti noodle (degree 20) follows every little bump — including noise.

5. The Bias-Variance Tradeoff

This is perhaps the most important concept in all of machine learning:

Total Error = Bias² + Variance + Irreducible Noise
  • Bias: error from overly simple models that miss the true pattern (underfitting).
  • Variance: error from overly complex models that are sensitive to training data (overfitting).
  • Irreducible noise: inherent randomness in the data you can't remove.

Simple models (low degree) have high bias, low variance. Complex models (high degree) have low bias, high variance. The sweet spot minimizes total error.

Regularization (Ridge, Lasso) is the primary tool for managing this tradeoff — it adds controlled bias to reduce variance.

6. Ridge Regression (L2 Regularization)

Ridge adds a penalty on the sum of squared weights to the loss:

Loss = Σ(yᵢ − ŷᵢ)² + λ · Σwⱼ²

The regularization parameter λ controls the penalty strength. As λ increases, weights shrink toward zero (but never exactly to zero).

w_ridge = (XᵀX + λI)⁻¹ Xᵀy
  • λ = 0: ordinary least squares (no regularization).
  • λ → ∞: all weights approach zero (underfitting).
  • Ridge keeps all features — it shrinks coefficients but doesn't eliminate any.
  • Great when you have many correlated features (stabilizes the solution).

7. Lasso Regression (L1 Regularization)

Lasso adds a penalty on the sum of absolute weights:

Loss = Σ(yᵢ − ŷᵢ)² + λ · Σ|wⱼ|

The key difference from Ridge: Lasso can push weights exactly to zero, performing automatic feature selection.

  • With enough λ, irrelevant features get eliminated entirely.
  • Produces sparse models — only the most important features remain.
  • No closed-form solution — must use iterative optimization (coordinate descent).
  • Can struggle when features are highly correlated (tends to pick one and drop others).
Ridge vs Lasso: Use Ridge when you believe all features matter (just want to shrink). Use Lasso when you suspect many features are irrelevant (want to select). Use Elastic Net when unsure.

8. Elastic Net

Elastic Net combines both L1 and L2 penalties:

Loss = Σ(yᵢ − ŷᵢ)² + λ₁·Σ|wⱼ| + λ₂·Σwⱼ²

Or equivalently with a mixing parameter α ∈ [0,1]:

Penalty = λ · [α·Σ|wⱼ| + (1−α)·Σwⱼ²]
  • α = 1: pure Lasso. α = 0: pure Ridge.
  • Best of both worlds: feature selection (from L1) + stability with correlated features (from L2).
  • Default choice when you don't know which regularization to use.

Always evaluate on a held-out test set. Training R² always increases with more features — use adjusted R² or cross-validation to get honest estimates.

10. Assumptions & Diagnostics

Linear regression assumes:

  1. Linearity: the relationship between X and y is linear. Check with residual plots.
  2. Independence: observations are independent of each other.
  3. Homoscedasticity: constant variance of residuals. Look for funnel shapes in residual plots.
  4. Normality: residuals are approximately normally distributed (important for confidence intervals).
  5. No multicollinearity: features are not too correlated with each other (check VIF).

If assumptions are violated: try transforming variables (log, sqrt), adding interaction terms, using robust regression, or switching to nonlinear models.

14. Summary & Key Takeaways

  1. Linear regression is the foundation — understand OLS, R², and the closed-form solution.
  2. Polynomial regression handles nonlinear patterns but easily overfits. Keep degrees low.
  3. The bias-variance tradeoff governs all model selection. Simple models underfit; complex ones overfit.
  4. Ridge (L2) shrinks all coefficients — great for correlated features.
  5. Lasso (L1) drives coefficients to zero — automatic feature selection.
  6. Elastic Net combines both — the safe default when unsure.
  7. Always validate on held-out data and check regression assumptions.
  8. Scale your features before applying regularization.
