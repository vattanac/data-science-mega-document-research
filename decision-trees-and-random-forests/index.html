1. What Is a Decision Tree?

A decision tree makes predictions by asking a series of yes/no questions about features, splitting data at each node until it reaches a prediction (leaf). It's like a flowchart for decision-making.

A decision tree is like playing 20 Questions. At each step, you ask the question that best narrows down the answer. "Is it bigger than a car?" splits the possibilities roughly in half. Each question partitions the remaining space until you arrive at the answer.
  • Root node: the first split (most informative feature).
  • Internal nodes: decision points that split data by a feature threshold.
  • Leaf nodes: final predictions (class label for classification, mean value for regression).
  • Depth: longest path from root to leaf.
Decision trees are the most interpretable ML model — you can explain every prediction as a series of rules: "if income > $50K AND age > 30 AND credit_score > 700 → approve loan."

2. Splitting Criteria

At each node, the tree picks the feature and threshold that best separates the classes. The "best" split maximizes information gain (or equivalently, reduces impurity the most).

Gini Impurity (used by CART, scikit-learn default)

Gini(S) = 1 − Σᵢ pᵢ²

Where pᵢ is the proportion of class i. Gini = 0 means pure (all one class). Gini = 0.5 means maximally mixed (50/50 for binary).

Entropy / Information Gain (used by ID3, C4.5)

Entropy(S) = −Σᵢ pᵢ log₂(pᵢ)
Information Gain = Entropy(parent) − Σ (nⱼ/n) · Entropy(childⱼ)

For Regression: Variance Reduction

Split quality = Var(parent) − Σ (nⱼ/n) · Var(childⱼ)

Choose the split that reduces variance the most. Leaf prediction = mean of remaining samples.

In practice, Gini and Entropy give nearly identical results. Gini is slightly faster to compute (no logarithm).

3. Building the Tree (CART Algorithm)

CART: Classification and Regression Trees

  1. Start with all data at the root.
  2. For each feature, try all possible thresholds.
  3. Pick the (feature, threshold) pair that maximizes information gain / minimizes impurity.
  4. Split data into left (≤ threshold) and right (> threshold) children.
  5. Recursively repeat on each child.
  6. Stop when: max depth reached, min samples per leaf, or node is pure.

Complexity: O(n · d · n log n) per level, where n = samples, d = features. Total: O(n · d · n log n · depth).

CART always produces binary splits (two children). For categorical features with K categories, it considers all 2^(K-1) − 1 possible binary partitions.

4. Overfitting & Pruning

An unrestricted tree will keep splitting until every leaf has one sample — perfectly memorizing training data but failing on new data. This is overfitting.

Pre-pruning (Stopping Criteria)

  • max_depth: limit tree depth (most important hyperparameter).
  • min_samples_split: minimum samples required to split a node.
  • min_samples_leaf: minimum samples in each leaf.
  • max_features: consider only a subset of features at each split.

Post-pruning (Cost-Complexity Pruning)

Grow the full tree, then prune back branches that don't improve cross-validated performance. Uses the parameter ccp_alpha — higher values produce smaller trees.

Cost = Σ (impurity at leaves) + α · |number of leaves|

5. Random Forests (Bagging)

A single tree is unstable — small changes in data produce very different trees. Random Forests fix this by training many trees and averaging their predictions.

How Random Forests Work

  1. Bootstrap: for each tree, draw a random sample (with replacement) from training data.
  2. Random feature subset: at each split, only consider √d features (classification) or d/3 (regression).
  3. Grow full tree: no pruning needed (individual trees overfit, but the ensemble generalizes).
  4. Aggregate: majority vote (classification) or average (regression) across all trees.
A Random Forest is like asking 100 different experts for advice, where each expert has seen a slightly different subset of evidence and focuses on different aspects. Individually they may be wrong, but their collective wisdom is usually right.

Why It Works

Each tree is a high-variance, low-bias estimator. Averaging many uncorrelated trees reduces variance without increasing bias. The randomness (bootstrap + feature subsets) ensures trees are different from each other.

  • Strengths: excellent accuracy, handles high-dimensional data, robust to outliers, rarely overfits, gives feature importance, works out-of-the-box.
  • Limitations: less interpretable than single tree, slow for very large ensembles, doesn't extrapolate (predictions bounded by training data range).

6. Gradient Boosting

While Random Forests train trees independently (in parallel), Gradient Boosting trains trees sequentially — each new tree corrects the errors of the previous ones.

Algorithm

  1. Start with a simple prediction (e.g., mean of y).
  2. Compute residuals (errors) of current model.
  3. Train a small tree to predict the residuals.
  4. Add this tree's predictions (scaled by learning rate) to the model.
  5. Repeat: each new tree learns from the remaining errors.
F_m(x) = F_{m-1}(x) + η · h_m(x) where h_m fits the residuals

The learning rate (η) controls how much each tree contributes. Smaller η = more trees needed = better generalization but slower training.

Boosting vs Bagging: Bagging (Random Forest) reduces variance by averaging independent trees. Boosting reduces bias by iteratively correcting errors. Boosting usually wins on accuracy but is more prone to overfitting.

7. XGBoost, LightGBM & CatBoost

Modern gradient boosting libraries with optimizations that make them the dominant choice for tabular data in competitions and industry.

LibraryKey InnovationSpeedBest For
XGBoostRegularized objective, column subsampling, sparsity-awareFastGeneral purpose, Kaggle
LightGBMLeaf-wise growth, histogram binning, GOSSVery fastLarge datasets, speed-critical
CatBoostOrdered boosting, native categorical handlingFastCategorical-heavy data

XGBoost Objective

Obj = Σ L(yᵢ, ŷᵢ) + Σ [γT + ½λ‖w‖²]

Loss function + regularization (γ controls tree complexity via number of leaves T, λ penalizes leaf weights w). This built-in regularization is why XGBoost generalizes better than vanilla gradient boosting.

8. Feature Importance

Tree-based models naturally provide feature importance — a huge advantage for interpretability.

  • Impurity-based (MDI): total reduction in Gini/entropy from splits on each feature, averaged across trees. Fast but biased toward high-cardinality features.
  • Permutation importance: shuffle each feature and measure accuracy drop. Unbiased, works for any model, but slower.
  • SHAP values: game-theoretic approach giving per-sample, per-feature attribution. Gold standard for interpretability.

14. Summary & Key Takeaways

  1. Decision trees are interpretable and intuitive, but a single tree overfits easily.
  2. Random Forests (bagging) average many independent trees — reducing variance with almost no tuning needed.
  3. Gradient Boosting trains sequential trees, each correcting the last — reducing bias for higher accuracy.
  4. XGBoost/LightGBM/CatBoost are the state-of-the-art for tabular data — regularized, fast, powerful.
  5. Feature importance (permutation or SHAP) makes tree models interpretable even as ensembles.
  6. Key hyperparameters: max_depth, learning_rate, n_estimators.
  7. For tabular data, tree ensembles often beat neural networks — try them first.
