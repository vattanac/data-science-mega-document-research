1. What Is Clustering?

Clustering is unsupervised learning — you have data with no labels, and you want to discover natural groups (clusters) within it. Points in the same cluster should be similar; points in different clusters should be dissimilar.

Imagine you dump a bag of mixed candies on a table. Without anyone telling you the types, you'd naturally group them by color, shape, and size. That's clustering — finding structure in unlabeled data.

Applications: customer segmentation, anomaly detection, image segmentation, document grouping, gene expression analysis, social network community detection.

Types of Clustering

  • Partitional: divide data into non-overlapping groups (K-Means, K-Medoids).
  • Density-based: groups are dense regions separated by sparse regions (DBSCAN, Mean Shift).
  • Hierarchical: nested tree of clusters at multiple granularities (Agglomerative, Divisive).
  • Model-based: fit statistical models to the data (Gaussian Mixture Models).

2. K-Means Clustering

K-Means is the most popular clustering algorithm. It partitions n points into K clusters by minimizing within-cluster sum of squares (WCSS/inertia).

Algorithm (Lloyd's Algorithm)

  1. Initialize: choose K random centroids (cluster centers).
  2. Assign: assign each point to the nearest centroid.
  3. Update: recompute each centroid as the mean of its assigned points.
  4. Repeat steps 2–3 until centroids stop moving (convergence).
Objective: min Σₖ Σᵢ∈Cₖ ‖xᵢ − μₖ‖²

Where μₖ is the centroid of cluster Cₖ. This is an NP-hard problem, but Lloyd's algorithm finds a good local minimum quickly.

Strengths & Limitations

  • Strengths: fast (O(nKd) per iteration), simple, scales well, works great for spherical clusters.
  • Limitations: must specify K in advance, sensitive to initialization, assumes spherical equal-size clusters, sensitive to outliers, finds only convex clusters.

3. K-Means++ Initialization

Bad initialization can lead to poor clustering. K-Means++ is a smarter initialization that spreads initial centroids apart:

  1. Pick the first centroid randomly from data points.
  2. For each remaining point, compute distance to nearest existing centroid.
  3. Pick next centroid with probability proportional to distance² (farther points more likely).
  4. Repeat until K centroids are chosen.
K-Means++ gives O(log K)-competitive results compared to optimal clustering, and converges faster. It's the default in scikit-learn and should always be used.

4. Choosing K: Elbow Method & Silhouette

Elbow Method

Plot WCSS (inertia) vs K. Look for the "elbow" — the point where adding more clusters stops significantly reducing inertia. The bend in the curve suggests the optimal K.

Silhouette Score

s(i) = (b(i) − a(i)) / max(a(i), b(i))

Where a(i) = average distance to points in same cluster, b(i) = average distance to points in nearest other cluster. Score ranges from −1 to +1:

  • +1: perfectly clustered (far from other clusters).
  • 0: on the boundary between clusters.
  • −1: probably in the wrong cluster.

Choose K that maximizes the average silhouette score.

5. DBSCAN (Density-Based Spatial Clustering)

DBSCAN finds clusters as dense regions separated by sparse regions. It doesn't require K and can find arbitrarily shaped clusters.

Key Concepts

  • ε (epsilon): neighborhood radius — how close points must be to be "neighbors."
  • minPts: minimum points needed in an ε-neighborhood to form a dense region.
  • Core point: has ≥ minPts neighbors within ε.
  • Border point: within ε of a core point but has < minPts neighbors itself.
  • Noise point: not within ε of any core point — an outlier.

Algorithm

  1. For each unvisited point, check if it's a core point.
  2. If core: start a new cluster. Add all density-reachable points (recursively expand through connected core points).
  3. Border points get assigned to a nearby cluster. Noise points remain unassigned.

Strengths & Limitations

  • Strengths: no K needed, finds arbitrary shapes, identifies outliers naturally, robust to noise.
  • Limitations: sensitive to ε and minPts, struggles with varying-density clusters, O(n²) without spatial indexing (O(n log n) with KD-tree).

6. Hierarchical Clustering

Builds a tree (dendrogram) of clusters at every level of granularity.

Agglomerative (Bottom-Up)

  1. Start: each point is its own cluster.
  2. Merge the two closest clusters.
  3. Repeat until one cluster remains.
  4. Cut the dendrogram at desired height to get K clusters.

Linkage Methods

LinkageDistance Between ClustersBehavior
SingleMin distance between any two pointsChaining effect, elongated clusters
CompleteMax distance between any two pointsCompact, spherical clusters
AverageAverage of all pairwise distancesBalanced compromise
Ward'sIncrease in total WCSS after mergeMinimizes variance, like K-Means
Ward's linkage is usually the best default — it produces compact, even-sized clusters similar to K-Means. Use single linkage only if you expect elongated or chain-like clusters.

7. Gaussian Mixture Models (GMM)

GMM assumes data comes from a mixture of K Gaussian distributions. Each cluster is modeled as a Gaussian with its own mean and covariance — allowing elliptical cluster shapes.

p(x) = Σₖ πₖ · N(x | μₖ, Σₖ)

Where πₖ = mixing weight (how big each cluster is), μₖ = mean, Σₖ = covariance matrix.

EM Algorithm

  1. E-step: compute probability each point belongs to each Gaussian (soft assignment).
  2. M-step: update means, covariances, and mixing weights using weighted data.
  3. Repeat until convergence (log-likelihood stops increasing).

Key difference from K-Means: GMM gives soft assignments (probabilities) rather than hard assignments. A point might be 70% cluster A, 30% cluster B. This captures uncertainty.

  • Strengths: elliptical clusters, soft assignments, principled probabilistic model, can use BIC/AIC for model selection.
  • Limitations: must specify K, sensitive to initialization, can overfit with too many parameters, struggles in high dimensions.

8. Mean Shift

Mean Shift finds clusters by iteratively shifting points toward regions of highest density (modes of the density function).

x ← Σᵢ K(xᵢ − x) · xᵢ / Σᵢ K(xᵢ − x)

Where K is a kernel function (usually Gaussian). Each point "climbs" the density gradient until it reaches a mode. Points converging to the same mode form a cluster.

  • Strengths: no K required, finds arbitrary shapes, automatic number of clusters.
  • Limitations: slow (O(n²) per iteration), bandwidth parameter is critical, doesn't scale well.

10. Practical Guide

Decision Flowchart

  1. Large dataset (>100K)? → K-Means or Mini-Batch K-Means.
  2. Need outlier detection? → DBSCAN.
  3. Non-spherical clusters? → DBSCAN or GMM.
  4. Want a hierarchy / dendrogram? → Agglomerative clustering.
  5. Need soft assignments? → GMM.
  6. Don't know K? → DBSCAN, Mean Shift, or use silhouette/elbow to find K.

Common Pitfalls

  • Always scale features before clustering (StandardScaler). Distance-based methods are sensitive to feature scales.
  • K-Means with bad K will force data into wrong number of groups.
  • DBSCAN with wrong ε either merges everything (too large) or labels everything as noise (too small). Use k-distance plot to find good ε.
  • High dimensions: distances become meaningless. Use PCA/UMAP first to reduce dimensions.
  • Evaluation: use silhouette score, Davies-Bouldin index, or domain knowledge. There's no single "right" answer in unsupervised learning.

14. Summary & Key Takeaways

  1. K-Means is fast and simple — the go-to for large datasets with spherical clusters.
  2. DBSCAN finds arbitrary shapes and detects outliers — great for spatial data.
  3. Hierarchical clustering gives you a dendrogram — useful when you want multiple granularities.
  4. GMM provides soft assignments and models elliptical clusters probabilistically.
  5. Always scale your features before clustering.
  6. Use silhouette score or elbow method to choose K.
  7. Reduce dimensions with PCA/UMAP first for high-dimensional data.
  8. There's no universally "best" algorithm — the right choice depends on your data shape, size, and goals.
