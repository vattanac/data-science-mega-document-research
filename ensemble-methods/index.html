Ensemble Methods

Combining models for superior performance — bagging, boosting, stacking & beyond

Machine Learning Bagging Boosting XGBoost Stacking

1. Why Ensembles Work

A single model has its weaknesses — it might overfit, underfit, or be biased in specific regions of the data. Ensemble methods combine multiple models to produce predictions that are more accurate and robust than any individual model.

The key insight: if individual models make different errors (they're "diverse"), their mistakes cancel out when aggregated. This is the "wisdom of crowds" for machine learning.

Bias-Variance Decomposition: Error = Bias² + Variance + Irreducible Noise Bagging → reduces Variance (averaging uncorrelated models) Boosting → reduces Bias (sequentially fixing errors)

2. Bagging (Bootstrap Aggregating)

Bagging trains multiple models on different bootstrap samples (random samples with replacement) and aggregates their predictions:

  1. Draw B bootstrap samples from the training data
  2. Train one model on each sample independently
  3. Aggregate: majority vote (classification) or average (regression)
f_bag(x) = (1/B) Σ f_b(x) (regression) f_bag(x) = mode{f_b(x)} (classification)

Each bootstrap sample contains ~63.2% of unique training examples (the rest are duplicates). The ~36.8% left out form the out-of-bag (OOB) samples — a free validation set!

Random Forest = Bagging + Feature Randomness

Random Forest adds another layer of diversity: at each split, only a random subset of features is considered (typically √p for classification, p/3 for regression). This decorrelates the trees, making the ensemble even more powerful.

3. Boosting — The Big Idea

Boosting trains models sequentially, where each new model focuses on the mistakes of the previous ones. The final prediction is a weighted combination of all models.

Unlike bagging (which reduces variance), boosting primarily reduces bias — it turns weak learners (slightly better than random) into a strong learner.

AdaBoost (Adaptive Boosting)

Adjusts sample weights: misclassified examples get higher weight so the next model pays more attention to them. Final prediction is a weighted vote, where better models get more influence.

Gradient Boosting

Instead of reweighting samples, each new model fits the residuals (errors) of the current ensemble. Equivalent to gradient descent in function space — hence the name.

5. AdaBoost in Detail

For m = 1, 2, ..., M: 1. Train weak learner h_m on weighted data 2. Compute weighted error: ε_m = Σ w_i · I(h_m(x_i) ≠ y_i) 3. Compute model weight: α_m = ½ ln((1 − ε_m) / ε_m) 4. Update sample weights: w_i ← w_i · exp(−α_m · y_i · h_m(x_i)) 5. Normalise weights Final: H(x) = sign(Σ α_m · h_m(x))

Models with lower error get higher α (more influence). Misclassified samples get higher weight for the next round. The algorithm is sensitive to outliers (they accumulate ever-increasing weight).

6. Gradient Boosting

Gradient Boosting generalises AdaBoost by fitting residuals using any differentiable loss function:

For m = 1, 2, ..., M: 1. Compute pseudo-residuals: r_i = −∂L(y_i, F_{m-1}(x_i)) / ∂F_{m-1}(x_i) 2. Fit a tree h_m to the residuals r_i 3. Update: F_m(x) = F_{m-1}(x) + η · h_m(x) η = learning rate (shrinkage, typically 0.01–0.3)

The learning rate η controls how much each tree contributes. Lower η needs more trees but generalises better — the classic trade-off.

7. XGBoost, LightGBM & CatBoost

XGBoost (Extreme Gradient Boosting)

Adds L1/L2 regularisation on leaf weights, uses second-order Taylor expansion of the loss, histogram-based split finding, and built-in handling of missing values. The Kaggle champion for years.

LightGBM

Microsoft's implementation using leaf-wise tree growth (vs XGBoost's level-wise), gradient-based one-side sampling (GOSS), and exclusive feature bundling (EFB). Much faster on large datasets.

CatBoost

Yandex's implementation with native categorical feature handling (ordered target encoding), ordered boosting to prevent target leakage, and symmetric tree structure for faster inference.

9. Stacking (Stacked Generalisation)

Stacking uses a meta-learner to combine predictions from multiple diverse base models:

  1. Train K diverse base models (e.g., Random Forest, SVM, Neural Net, XGBoost)
  2. Generate out-of-fold predictions from each base model
  3. Train a meta-learner (often logistic regression or linear model) on these predictions
  4. Final prediction = meta-learner(base_model_predictions)

The meta-learner learns the optimal way to combine base models — which to trust and when. Cross-validation is critical to avoid overfitting the meta-learner to the training set.

10. Voting Ensembles

Hard Voting

Each model casts a vote; the majority wins. Simple but effective when models have similar accuracy.

Soft Voting

Average the predicted probabilities, then pick the class with highest average probability. Usually better than hard voting because it uses confidence information.

Hard: ŷ = mode(h₁(x), h₂(x), ..., hₖ(x)) Soft: ŷ = argmax_c (1/K) Σ P_k(y=c|x)

13. When to Use What

  • Random Forest: Great default. Low tuning effort, resistant to overfitting, handles mixed feature types well.
  • XGBoost/LightGBM: When you need maximum accuracy and are willing to tune. Kaggle competitions, tabular data, production ML pipelines.
  • CatBoost: When you have many categorical features and want minimal preprocessing.
  • AdaBoost: Simple baseline for boosting; works well with decision stumps.
  • Stacking: When you have diverse strong models and want to squeeze out the last bit of accuracy.
  • Voting: Quick win when you already have multiple trained models.
