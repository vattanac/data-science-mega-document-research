Activation Functions

Deep Learning's Hidden Engine

DEEP LEARNING NEURAL NETWORKS

Why Activation Functions?

The Problem: Linear → Linear

Without activation functions, neural networks are just a series of matrix multiplications:

y = W₃(W₂(W₁x + b₁) + b₂) + b₃

This simplifies to a single linear transformation:

y = Wx + b
Problem: No matter how many layers you stack, linear functions composed are still linear. This means your neural network can only learn linear relationships—exactly what logistic regression already does!

The Solution: Non-Linearity

Activation functions introduce non-linearity, allowing networks to learn complex, curved decision boundaries and capture intricate patterns in data.

h₁ = σ(W₁x + b₁) h₂ = σ(W₂h₁ + b₂) y = σ(W₃h₂ + b₃)

Where σ is a non-linear activation function. Now the network has the expressiveness to learn any function.

Key Insight: Activation functions are the bridge between linear algebra (what computers do fast) and non-linear learning (what we need for intelligence).

Sigmoid

Formula

σ(x) = 1 / (1 + e^(-x))

Maps any input to a value between 0 and 1. Historically the most popular.

Properties

Range: (0, 1)

Derivative: σ'(x) = σ(x)(1 - σ(x))

Maximum derivative: 0.25 (at x=0)

Visualization

Characteristics

Pros

  • Smooth gradient everywhere
  • Historically proven
  • Perfect for binary classification
  • Interpretable as probability

Cons

  • Vanishing gradient problem
  • Not zero-centered (slow convergence)
  • Computationally expensive (exp)
  • Gradient nearly zero at extremes

The Vanishing Gradient Problem

Issue: When sigmoid outputs approach 0 or 1, the gradient becomes nearly zero. In deep networks, gradients multiply through layers using backpropagation, so small gradients compound—by the time you reach early layers, the gradient is infinitesimally small and training slows to a halt.

This is why sigmoid largely disappeared from hidden layers in modern deep networks, though it's still used in output layers for binary classification.

Tanh (Hyperbolic Tangent)

Formula

tanh(x) = (e^x - e^(-x)) / (e^x + e^(-x))

Or more conveniently: tanh(x) = 2σ(2x) - 1

Properties

Range: (-1, 1)

Derivative: tanh'(x) = 1 - tanh²(x)

Maximum derivative: 1 (at x=0, vs sigmoid's 0.25)

Visualization

Key Advantage: Zero-Centered

Better Gradient Flow: Tanh is zero-centered, meaning on average its outputs are near 0. This reduces zig-zagging during gradient descent and leads to faster convergence compared to sigmoid.

Characteristics

Pros

  • Zero-centered output
  • Stronger gradients than sigmoid
  • Better for hidden layers
  • Smooth differentiation

Cons

  • Still suffers vanishing gradient
  • Computationally expensive
  • Less commonly used now

Tanh addresses sigmoid's symmetry problem but still faces vanishing gradients. In modern architectures (2010s onwards), it's been largely superseded by ReLU and variants.

ReLU (Rectified Linear Unit)

Formula

ReLU(x) = max(0, x)

The simplest non-linearity: if x > 0, output x; otherwise output 0.

Properties

Range: [0, ∞)

Derivative: 1 if x > 0, else 0

Crucially, no vanishing gradient (gradient is constant 1 for positive inputs)

Visualization

Why ReLU Revolutionized Deep Learning

Game-Changer: ReLU's constant gradient for positive values solved the vanishing gradient problem, enabling training of very deep networks (100+ layers). This was pivotal for modern deep learning.

Characteristics

Pros

  • No vanishing gradient
  • Computationally very cheap
  • Encourages sparse activations
  • Empirically works well

Cons

  • Dying ReLU problem
  • Not differentiable at 0
  • Unbounded activations

The Dying ReLU Problem

Critical Issue: If a neuron's output becomes negative during training, the gradient becomes 0 and stays 0 forever—the neuron is "dead" and will never activate again. Once a large portion of neurons die, the network effectively loses capacity.

Cause: Bad initialization or high learning rates can push weights such that all inputs to a neuron are negative. Subsequent gradient updates can't fix this because the gradient is zero.

Leaky ReLU & ELU

Leaky ReLU

LeakyReLU(x) = x if x > 0, else αx (typically α = 0.01)

Allows a small negative slope instead of a hard zero.

Properties

Range: (-∞, ∞)

Derivative: 1 if x > 0, else α

Gradient is never exactly zero, preventing dead neurons

Visualization

Advantage: Solves the dying ReLU problem by maintaining gradient flow even for negative inputs. Dead neurons can still learn and recover.

ELU (Exponential Linear Unit)

ELU(x) = x if x > 0, else α(e^x - 1) (typically α = 1.0)

Smoother version of Leaky ReLU with exponential behavior for negative inputs.

Properties

Range: (-α, ∞)

Derivative: 1 if x > 0, else αe^x

Smoother and closer to zero-centered

Visualization

Advantage: Combines the benefits of ReLU (fast for positive) with smoother gradients. Outputs closer to zero-mean, aiding convergence.

Comparison: ReLU vs Leaky vs ELU

ReLU: Fast and simple, but dies. Use when you have careful initialization and regularization.

Leaky ReLU: Practical improvement over ReLU, low computational overhead. Good default choice.

ELU: Smoother gradient, better convergence. Slightly more expensive due to exponential. Great for harder problems.

Softmax

Formula

softmax(z)ᵢ = e^(zᵢ) / Σⱼ e^(zⱼ)

Converts a vector of logits into a probability distribution over classes.

Key Properties

Range: (0, 1) for each output, sum = 1

Use Case: Multi-class classification (output layer)

Interpretation: Each output is the probability of that class

Example

Input: [2.0, 1.0, 0.1] e^2.0 ≈ 7.39 e^1.0 ≈ 2.72 e^0.1 ≈ 1.11 Sum ≈ 11.22 Output: [7.39/11.22, 2.72/11.22, 1.11/11.22] ≈ [0.659, 0.242, 0.099]

Important Details

Numerically Stable Implementation: For numerical stability, subtract the max value from inputs before computing softmax:
softmax(z)ᵢ = e^(zᵢ - max(z)) / Σⱼ e^(zⱼ - max(z))
This prevents overflow since e^large becomes infinity.

Characteristics

Pros

  • Outputs valid probability distribution
  • Clear probabilistic interpretation
  • Differentiable everywhere
  • Works with cross-entropy loss

Cons

  • Only for output layer (multi-class)
  • Can have softmax mismatch with other losses
  • Slightly expensive (exp for all classes)

In Practice

Softmax is typically paired with cross-entropy loss for multi-class classification. Most frameworks (PyTorch, TensorFlow) provide numerically stable combined implementations.

Key Takeaways

1. Non-linearity is Everything: Without activation functions, your entire network collapses to linear regression. Activation functions give neural networks their power.
2. Modern Default: ReLU & Variants: For hidden layers, use ReLU (or Leaky ReLU/ELU if having dead neuron issues). It's fast, effective, and proven at scale.
3. Choose Output Based on Task:
  • Binary classification → Sigmoid
  • Multi-class → Softmax
  • Regression → Linear (no activation)
4. Avoid Sigmoid/Tanh for Hidden Layers: Vanishing gradients make training deep networks painful. They're historical relics in hidden layers.
5. Dying ReLU is Real: If training stalls, check if neurons are dying. Leaky ReLU or ELU are cheap fixes.
