Hypothesis Testing

Understanding T-Tests, P-Values, and Statistical Decision Making

What Is Hypothesis Testing?

Hypothesis testing is a statistical method used to make decisions about populations based on sample data. It provides a structured framework for evaluating whether observed differences or relationships are statistically significant or simply due to random chance.

Core Concept: We start by assuming a "null hypothesis" (H₀) is true, then ask: "How likely is our observed data if H₀ is really true?" If the answer is "very unlikely," we reject H₀ in favor of the "alternative hypothesis" (H₁).

The Two Hypotheses

Null Hypothesis (H₀)
Assumes no effect, no difference, or no relationship exists. It's the "default" position we start with. Example: "The average height of men and women is the same."
Alternative Hypothesis (H₁)
Contradicts the null hypothesis. It represents what we're trying to find evidence for. Example: "The average height of men and women is different."
Key Insight: Hypothesis testing never "proves" H₁ is true; it only decides whether there's enough evidence to reject H₀. If we reject H₀, we have statistical support for H₁.

The Five Steps of Hypothesis Testing

  1. State the Hypotheses

    Define H₀ and H₁ based on your research question. Be clear about whether you're testing for equality, difference, greater than, or less than.

  2. Choose a Significance Level (α)

    Select your "cutoff" probability for rejecting H₀. Common values are α = 0.05 (5%) or α = 0.01 (1%). This is your tolerance for "Type I Error."

  3. Compute the Test Statistic

    Calculate the appropriate statistic (t-statistic, z-score, chi-square, etc.) based on your data and the test you're using.

  4. Find the P-Value

    Determine the probability of observing data as extreme (or more extreme) than what you observed, assuming H₀ is true.

  5. Make a Decision

    Compare p-value to α. If p-value ≤ α, reject H₀. If p-value > α, fail to reject H₀.

Type I Error (α): Rejecting H₀ when it's actually true. This is a "false alarm" or "false positive." We control this by setting our significance level.
Type II Error (β): Failing to reject H₀ when it's actually false. This is a "missed detection." Reducing β requires larger sample sizes or more powerful tests.
Power (1 - β): The probability of correctly rejecting H₀ when it's false. Higher power is better. Power depends on sample size, effect size, and significance level.

When to Use Which Test

Test When to Use Conditions Output
Z-Test Compare one sample to a population with known σ σ known, large n (n ≥ 30) Z-score, p-value
T-Test Compare means of 1 or 2 samples with unknown σ σ unknown, small to medium n T-statistic, p-value, df
Paired T-Test Compare two related samples (before/after) Observations are paired, independent pairs T-statistic, p-value
Chi-Square Test independence or goodness-of-fit for categorical data Categorical variables, expected freq ≥ 5 χ² statistic, p-value, df
ANOVA Compare means across 3+ groups Independent groups, approximately normal, equal variances F-statistic, p-value
Rule of Thumb: Use t-test for comparing 2 group means, ANOVA for 3+ groups, chi-square for categorical associations, and z-test only when population standard deviation is known.

Why Hypothesis Testing Matters in Data Science

A/B Testing & Experimentation

Before rolling out a new feature or change, data scientists run A/B tests to determine if the difference in conversion rates, engagement, or other metrics is statistically significant or just random noise. Hypothesis testing ensures decisions are data-driven.

Feature Selection

When building models, we test whether each feature has a statistically significant relationship with the target variable. Features with low p-values are kept; those with high p-values may be dropped to improve model simplicity and reduce overfitting.

Model Comparison

Comparing two models? Hypothesis testing helps determine if one truly outperforms the other, or if differences are due to chance. This prevents us from choosing a model that just got lucky on a particular test set.

Quality Assurance & Monitoring

In production systems, we monitor key metrics and use hypothesis tests to detect when performance deviates significantly from expected levels, triggering alerts for investigation.

The Bottom Line: Hypothesis testing protects data scientists from being misled by random variation. It's the bridge between data and confident decision-making.
