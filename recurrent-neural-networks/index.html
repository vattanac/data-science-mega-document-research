Recurrent Neural Networks

Sequence modelling — from vanilla RNNs to LSTMs, GRUs & Seq2Seq

Deep Learning Sequences LSTM GRU Seq2Seq

1. Why Sequences Need Special Architectures

Standard feedforward networks treat each input independently — they have no notion of order or memory. But many real-world problems are sequential: language, audio, time series, video, DNA.

A sentence's meaning depends on word order ("dog bites man" vs "man bites dog"). A stock price depends on its history. We need architectures that can remember previous inputs and use that context to process the current one.

Recurrent Neural Networks solve this by introducing loops — the output at each timestep feeds back as input to the next, creating a form of memory.

2. The Vanilla RNN

At each timestep t, the RNN takes input x(t) and the previous hidden state h(t−1), producing a new hidden state:

h(t) = tanh(W_hh · h(t−1) + W_xh · x(t) + b_h) y(t) = W_hy · h(t) + b_y

The hidden state h(t) is the network's "memory" — a compressed representation of everything it has seen so far. The same weights W_hh, W_xh, W_hy are shared across all timesteps (parameter sharing).

Unrolling Through Time

To understand training, we "unroll" the RNN: duplicate the network for each timestep, creating a very deep feedforward network where each layer shares weights. This is why RNNs can handle variable-length sequences.

3. Backpropagation Through Time (BPTT)

Training an RNN uses BPTT — backpropagation applied to the unrolled network. The loss is summed across timesteps, and gradients flow backwards through the entire sequence.

L = Σ L(t) (total loss across timesteps) ∂L/∂W = Σ ∂L(t)/∂W (gradients accumulated)

The Vanishing Gradient Problem

Gradients are multiplied by W_hh at each timestep. If the largest eigenvalue of W_hh is < 1, gradients shrink exponentially — the network "forgets" long-range dependencies. If > 1, gradients explode.

∂h(t)/∂h(k) = Π_{i=k+1}^{t} diag(tanh'(·)) · W_hh For t − k large: gradients → 0 (vanishing) or → ∞ (exploding)

Gradient clipping handles exploding gradients (cap the norm). But vanishing gradients require architectural solutions — enter LSTM and GRU.

5. LSTM — Long Short-Term Memory

LSTMs (Hochreiter & Schmidhuber, 1997) solve vanishing gradients by introducing a cell state c(t) — a highway that carries information across timesteps with minimal interference — controlled by three gates:

Forget Gate (f)

Decides what to discard from the cell state: f(t) = σ(W_f · [h(t−1), x(t)] + b_f)

Output is between 0 (forget everything) and 1 (keep everything).

Input Gate (i) + Candidate

Decides what new information to store: i(t) = σ(W_i · [h(t−1), x(t)] + b_i)

Candidate values: c̃(t) = tanh(W_c · [h(t−1), x(t)] + b_c)

Cell State Update

c(t) = f(t) ⊙ c(t−1) + i(t) ⊙ c̃(t)

The cell state is a linear combination — gradients flow through with minimal decay!

Output Gate (o)

Decides what to output: o(t) = σ(W_o · [h(t−1), x(t)] + b_o)

Hidden state: h(t) = o(t) ⊙ tanh(c(t))

6. GRU — Gated Recurrent Unit

GRUs (Cho et al., 2014) simplify the LSTM to two gates — fewer parameters, often comparable performance:

Update gate: z(t) = σ(W_z · [h(t−1), x(t)]) Reset gate: r(t) = σ(W_r · [h(t−1), x(t)]) Candidate: h̃(t) = tanh(W · [r(t) ⊙ h(t−1), x(t)]) Hidden state: h(t) = (1 − z(t)) ⊙ h(t−1) + z(t) ⊙ h̃(t)

The update gate z blends old and new state (like a combined forget+input gate). The reset gate r controls how much past state to expose when computing the candidate. No separate cell state — the hidden state does everything.

Gates3 (forget, input, output)2 (update, reset)

8. Bidirectional RNNs

Standard RNNs only see the past. A Bidirectional RNN runs two separate RNNs — one forward, one backward — and concatenates their hidden states:

h_fwd(t) = RNN_fwd(x(t), h_fwd(t−1)) h_bwd(t) = RNN_bwd(x(t), h_bwd(t+1)) h(t) = [h_fwd(t); h_bwd(t)] (concatenation)

This gives each position access to both left and right context. Essential for tasks like NER and sentiment analysis where future context matters. Used in ELMo, BERT's training (via masking), and many sequence labelling tasks.

9. Sequence-to-Sequence (Seq2Seq)

Seq2Seq models map variable-length input sequences to variable-length output sequences using an encoder-decoder architecture:

Encoder

Processes the input sequence and compresses it into a fixed-size context vector (the final hidden state).

Decoder

Takes the context vector and generates the output sequence one token at a time, autoregressively.

Applications: machine translation, text summarisation, chatbots, code generation. The bottleneck of compressing everything into a single vector led to the invention of attention mechanisms.

10. Attention in RNNs

Instead of relying on a single context vector, attention lets the decoder look at all encoder hidden states and focus on the most relevant ones at each decoding step:

score(s_t, h_i) = s_t^T · h_i (dot-product attention) α_i = softmax(score) (attention weights) context = Σ α_i · h_i (weighted sum)

This solved the information bottleneck and dramatically improved machine translation. Bahdanau attention (2014) used a learned alignment; Luong attention (2015) simplified it with dot products.

12. Practical Tips

  • Start with LSTM or GRU — never vanilla RNN for real tasks. GRU if speed matters; LSTM for longer sequences.
  • Gradient clipping — always clip gradients (max_norm=1.0 or 5.0) to prevent explosions.
  • Layer stacking — 2–3 layers is common. More layers need residual connections.
  • Dropout — apply between layers, NOT within recurrent connections. Use variational dropout for recurrent dropout.
  • Bidirectional — use for classification/labelling tasks; NOT for generation (can't see the future while generating).
  • Teacher forcing — during training, feed ground truth as decoder input; at inference, feed predicted tokens.
  • Consider Transformers — for most tasks today, transformers outperform RNNs. Use RNNs when: sequences are very long, latency matters, or you need online/streaming processing.

Modern hybrids like Mamba (state-space models) combine the best of both: linear-time processing with selective state updates, achieving transformer-level quality with RNN-like efficiency.
