Dot Product & Cosine Similarity

Understanding vector relationships in mathematics and machine learning

LINEAR ALGEBRA NLP

What Is the Dot Product?

The dot product (also called scalar product or inner product) is a fundamental operation that takes two vectors and returns a single scalar value. It measures the extent to which two vectors align in the same direction.

Definition

Given two vectors a = [a₁, a₂, ..., aₙ] and b = [b₁, b₂, ..., bₙ], the dot product is:

a · b = Σ(aᵢ × bᵢ) = a₁b₁ + a₂b₂ + ... + aₙbₙ

This can also be expressed using the geometric interpretation:

a · b = |a| × |b| × cos(θ)

Where |a| and |b| are the magnitudes (lengths) of the vectors, and θ is the angle between them.

Geometric Interpretation: The dot product tells us how much one vector "projects" onto another. When vectors point in the same direction (θ = 0°), the dot product is maximized. When they're perpendicular (θ = 90°), it's zero. When they point opposite (θ = 180°), it's negative.

Key Properties

  • Commutative: a · b = b · a
  • Distributive: a · (b + c) = a · b + a · c
  • With scalar k: (ka) · b = k(a · b)
  • Self-dot product: a · a = |a|² (always non-negative)

What Is Cosine Similarity?

Cosine similarity is a normalized measure derived from the dot product. It measures the angle (direction) between two vectors, independent of their magnitudes.

Definition

The cosine similarity between vectors a and b is:

cos(θ) = (a · b) / (|a| × |b|)

Or equivalently:

cos(θ) = Σ(aᵢ × bᵢ) / √(Σaᵢ²) × √(Σbᵢ²)
Range: Cosine similarity always returns a value between -1 and +1:
  • +1: Vectors point in the same direction (identical orientation)
  • 0: Vectors are perpendicular (orthogonal)
  • -1: Vectors point in opposite directions

Why Normalize?

By dividing by the magnitudes, we remove the effect of vector length. This is crucial for applications like text similarity, where we care about the direction of vectors (how similar the content is) rather than their magnitude (how long the text is).

Example: Two documents with identical content but different lengths should have high similarity, which cosine similarity achieves but raw dot product would not.

Dot Product vs Cosine Similarity

While related, these measures have different use cases and behaviors:

Dot Product

Range: -∞ to +∞ (unbounded)

Affected by: Direction AND magnitude

Use when: You care about the combined effect of direction and length (e.g., work done by a force, signal processing)

Example: Computing energy or power in physics

Cosine Similarity

Range: -1 to +1 (bounded)

Affected by: Direction only (magnitude independent)

Use when: You care about similarity of direction (e.g., document comparison, embeddings)

Example: Finding similar articles or products regardless of content length

Key Insight: If you have two vectors with identical direction but different lengths, the dot product will differ (proportional to the length), but cosine similarity will be exactly 1 (perfect match). This makes cosine similarity ideal for comparing content where scale shouldn't matter.

Effect of Magnitude

Consider vectors a = [1, 0] and b = [2, 0] (same direction):

  • Dot product: a · b = 1×2 + 0×0 = 2
  • Cosine similarity: cos(θ) = 2 / (1 × 2) = 1 (perfect alignment)

Now compare with c = [0.5, 0]:

  • Dot product: a · c = 1×0.5 = 0.5 (different from a·b)
  • Cosine similarity: cos(θ) = 0.5 / (1 × 0.5) = 1 (same result!)

Applications

Dot product and cosine similarity are foundational concepts used across countless fields:

📝 NLP & Word Embeddings

Measure semantic similarity between words or documents. Word2Vec, GloVe, and modern transformers use cosine similarity to find similar words in embedding space.

🛍️ Recommendation Systems

Find similar products, movies, or users. Cosine similarity of user-item interaction vectors recommends content aligned with user preferences.

🔍 Search Engines

Rank search results by relevance. Queries and documents are converted to vectors; cosine similarity determines how well documents match user queries.

🖼️ Image Similarity

Compare image features. Vectors extracted from images (via CNNs or other methods) can be compared using cosine similarity to find visually similar images.

⚡ Neural Network Attention

Attention mechanisms in transformers compute dot products between query and key vectors to determine which parts of input matter most.

📊 Dimensionality Reduction

Techniques like Principal Component Analysis use dot products and cosine similarity to project high-dimensional data into lower dimensions.

Real-World Example: When you search Google for "machine learning best practices," the search engine converts your query and all indexed documents into vectors. Cosine similarity between your query vector and each document vector determines the ranking. Documents with high cosine similarity (same direction) appear first.
