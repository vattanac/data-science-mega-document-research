Decision Trees & Random Forest

A comprehensive guide to tree-based machine learning models

What Is a Decision Tree?

A Decision Tree is a supervised learning algorithm that makes decisions by recursively splitting data into subsets based on feature values. It's one of the most interpretable and widely-used machine learning models.

Key Components

  • Root Node: The top node containing all training data
  • Internal Nodes: Decision nodes that split data based on feature thresholds
  • Leaves: Terminal nodes that contain class predictions (classification) or continuous values (regression)
  • Branches: Connections representing the outcomes of splits

Classification vs Regression

Classification Trees

Predict discrete class labels (e.g., spam/not spam, cat/dog/bird)

Uses metrics like Gini Impurity or Entropy to measure purity of splits

Regression Trees

Predict continuous numerical values (e.g., house prices, temperature)

Uses Mean Squared Error (MSE) to minimize prediction variance

Why Decision Trees? They're highly interpretable, require minimal data preprocessing, handle both numerical and categorical features, and naturally capture non-linear relationships without feature scaling.

How Splitting Works

The core of decision tree learning is finding the best splits that separate data by class. Two main metrics guide this process:

Gini Impurity

Measures the probability of incorrectly classifying a randomly chosen element if labeled according to the distribution of classes in the node.

Gini = 1 - Σ(p_i)² where p_i is the proportion of class i in the node Example: If a node has 10 samples (6 class A, 4 class B): Gini = 1 - (0.6² + 0.4²) = 1 - (0.36 + 0.16) = 0.48

Information Gain & Entropy

Information Gain measures how much a split reduces entropy (disorder) in the data.

Entropy = -Σ(p_i × log₂(p_i)) where p_i is the proportion of class i Information Gain = Entropy(parent) - Σ(weight × Entropy(child))

The algorithm chooses the split that maximizes Information Gain or minimizes Gini impurity across child nodes.

Greedy Algorithm: Trees use a greedy approach—at each node, they select the feature and threshold that best improves purity, without guaranteeing a globally optimal tree. This makes the algorithm computationally efficient but potentially suboptimal.

Overfitting & Pruning

Decision trees are prone to overfitting—growing too complex and memorizing the training data rather than learning generalizable patterns. Two strategies prevent this:

Pre-Pruning (Early Stopping)

Stop tree growth before it becomes too complex by setting constraints:

  • Max Depth: Limit how deep the tree can grow (e.g., max_depth=5)
  • Min Samples Split: Require a minimum number of samples to split a node (e.g., min_samples_split=10)
  • Min Samples Leaf: Require a minimum number of samples in each leaf node
  • Max Features: Limit the number of features considered at each split

Post-Pruning (Cost-Complexity Pruning)

Grow the full tree, then remove branches that don't significantly improve performance on validation data. This method often finds better trees than pre-pruning.

The Bias-Variance Trade-off: Pre-pruning adds bias (underfitting) but reduces variance. Post-pruning requires validation data but can better balance this trade-off. In practice, ensemble methods like Random Forests handle overfitting by averaging many trees.
Underfitting (Too Simple)

High bias, low variance

Model misses important patterns; poor performance on train and test data

Overfitting (Too Complex)

Low bias, high variance

Model memorizes noise; great on train, poor on test data

Random Forest

Random Forest is an ensemble learning method that combines hundreds of decision trees to make more robust and accurate predictions than a single tree.

How It Works

  1. Bagging (Bootstrap Aggregating): Create multiple random samples of the training data (with replacement). Each sample has the same size as the original data but contains duplicates.
  2. Grow Diverse Trees: Train an independent decision tree on each bootstrap sample.
  3. Feature Randomness: At each split, consider only a random subset of features (usually √n or log₂(n) features). This increases tree diversity.
  4. Aggregate Predictions: For classification, use majority voting. For regression, average predictions.

Why It's Better Than Single Trees

Single Tree
  • Interpretable
  • Fast to train
  • Prone to overfitting
  • High variance
Random Forest
  • Less interpretable
  • Slower to train/predict
  • Better generalization
  • Low variance, low bias
Out-of-Bag (OOB) Error: On average, each bootstrap sample excludes ~37% of the data. These "out-of-bag" samples serve as a validation set, allowing accurate performance estimation without a separate holdout set.

Feature Importance Visualization

Feature importance measures how much each feature contributes to the predictions in a tree or forest. Higher importance means the feature is more useful for making splits.

How It's Calculated

For each feature, sum up the improvement (information gain or Gini decrease) across all splits that use that feature, weighted by the number of samples affected. In Random Forests, average importance across all trees.

Applications in the Real World

Fraud Detection

Decision Trees and Random Forests analyze transaction features (amount, location, merchant type, time) to classify transactions as fraudulent or legitimate. The ensemble nature of Random Forests reduces false positives while maintaining high fraud detection rates.

Medical Diagnosis

Healthcare systems use tree-based models to predict disease risk based on patient features (age, blood pressure, cholesterol, family history). The interpretability of decision trees is especially valuable in medical domains where clinicians need to understand how diagnoses are made.

Credit Scoring

Banks employ Random Forests to predict loan default probability using applicant data (income, credit history, debt levels, employment status). These models balance predictive accuracy with regulatory compliance requirements for explainability.

Customer Segmentation

E-commerce platforms use tree-based models to classify customers into segments (high-value, at-risk, dormant) for targeted marketing campaigns based on purchase history, browsing behavior, and engagement metrics.

Recommendation Systems

Decision Trees help predict user preferences for movies, products, or content by analyzing historical interactions, demographics, and user behavior patterns.

Anomaly Detection

Isolation Forests (a variant) identify unusual patterns in network traffic, sensor data, or system logs by isolating anomalies in decision tree structures more efficiently than normal points.

Advantages Across Domains: Fast inference (important for real-time applications), interpretability (crucial for regulated industries), handling mixed data types, and no feature scaling requirements make tree-based models widely applicable.
