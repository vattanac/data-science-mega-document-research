1. What Is NLP?

Natural Language Processing is the field at the intersection of linguistics, computer science, and machine learning that gives computers the ability to understand, interpret, and generate human language.

Core NLP Tasks

  • Text Classification: Spam detection, sentiment analysis, topic labelling
  • Named Entity Recognition (NER): Identifying people, places, organisations in text
  • Machine Translation: Translating between languages
  • Question Answering: Extracting answers from context passages
  • Text Generation: Producing coherent text (chatbots, summaries, stories)
  • Semantic Similarity: Measuring how similar two texts are in meaning

2. Tokenization

Tokenization is the first step — breaking raw text into discrete units (tokens) that a model can process.

Word-level

Split on whitespace and punctuation. Simple but creates huge vocabularies and can't handle unseen words (OOV problem).

Character-level

Each character is a token. Tiny vocabulary (~100) but sequences become very long, making it hard for models to learn long-range dependencies.

Subword (BPE / WordPiece / Unigram)

The sweet spot. Common words stay whole; rare words are split into meaningful subword units. "unhappiness" → ["un", "##happiness"]. Used by GPT (BPE), BERT (WordPiece), and T5 (Unigram).

4. Word Embeddings

Embeddings map tokens to dense vectors in a continuous space where semantic relationships are preserved.

Word2Vec (2013)

Two architectures: Skip-gram (predict context from word) and CBOW (predict word from context). Captures relationships like king − man + woman ≈ queen.

GloVe (2014)

Global Vectors — learns embeddings from word co-occurrence statistics across the entire corpus. Combines the benefits of global matrix factorisation and local context windows.

Contextual Embeddings (ELMo, BERT)

Unlike static embeddings, contextual embeddings produce different vectors for the same word depending on context. "Bank" in "river bank" vs "bank account" gets different representations.

Embedding dimension typically: 100–768 Word2Vec/GloVe: static, one vector per word ELMo/BERT: contextual, vector depends on surrounding words

5. Classical Representations: BoW & TF-IDF

Before deep learning, text was represented using count-based methods:

Bag of Words (BoW)

Each document is a vector of word counts. Ignores word order entirely — "dog bites man" and "man bites dog" get the same representation.

TF-IDF

TF-IDF(t, d) = TF(t, d) × IDF(t) TF(t, d) = count of term t in document d / total terms in d IDF(t) = log(N / df(t)) where df(t) = docs containing t

TF-IDF downweights common words ("the", "is") and upweights distinctive words. Still widely used as a baseline and for feature engineering.

6. Recurrent Neural Networks for NLP

RNNs process sequences token by token, maintaining a hidden state that captures context:

h(t) = tanh(W_h · h(t−1) + W_x · x(t) + b)

The hidden state h(t) is a compressed summary of all tokens seen so far. However, vanilla RNNs suffer from the vanishing gradient problem — they struggle to learn long-range dependencies.

LSTM & GRU

LSTM (Long Short-Term Memory) adds gates — forget, input, output — that control information flow. GRU (Gated Recurrent Unit) simplifies to two gates — reset and update. Both solve the vanishing gradient problem.

Bidirectional variants process text in both directions, capturing both left and right context.

7. The Attention Mechanism

Attention allows a model to focus on different parts of the input when producing each output. Instead of compressing everything into a fixed-size vector, attention computes a weighted sum over all input positions.

Attention(Q, K, V) = softmax(Q K^T / √d_k) V Q = queries, K = keys, V = values d_k = dimension of keys (scaling factor)

The softmax(QK^T/√d_k) produces an attention weight matrix — each entry says "how much should position i attend to position j?"

Self-Attention

When Q, K, and V all come from the same sequence, it's called self-attention. Every token attends to every other token, capturing relationships regardless of distance.

9. The Transformer Architecture

The Transformer (Vaswani et al., 2017) replaced RNNs with pure attention. It processes all tokens in parallel, making it vastly more efficient for training on GPUs.

Encoder

Stack of N identical layers, each with: (1) Multi-Head Self-Attention, (2) Feed-Forward Network. Plus residual connections and layer normalisation. Used by BERT.

Decoder

Similar to encoder but with masked self-attention (can only attend to previous positions) and cross-attention to encoder output. Used by GPT (decoder-only) and T5 (encoder-decoder).

Multi-Head Attention

Instead of one attention function, split Q/K/V into h heads, each attending to different aspects. Concatenate and project. Typical: h=8 or 12 heads.

Positional Encoding

Since transformers have no inherent notion of order, positional information is injected:

PE(pos, 2i) = sin(pos / 10000^(2i/d_model)) PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))

Fine-Tuning vs Prompting

Fine-tuning: Train the pre-trained model further on your specific task with labelled data. Best for specialised tasks.

Prompting / In-Context Learning: Give the model examples in the prompt without updating weights. Powerful for large models (GPT-4, Claude).

11. Text Classification Pipeline

A typical modern text classification pipeline:

  1. Tokenize — Convert text to token IDs using the model's tokenizer
  2. Encode — Pass through pre-trained model to get contextual embeddings
  3. Pool — Extract a single vector (e.g., [CLS] token for BERT, last token for GPT)
  4. Classify — Pass through a linear layer + softmax for class probabilities
  5. Fine-tune — Train end-to-end on labelled examples with cross-entropy loss
Input: "This movie was fantastic!" → Tokens: [CLS] this movie was fantastic ! [SEP] → BERT Encoder → [CLS] embedding (768-d vector) → Linear(768, 2) → softmax → [0.02, 0.98] → Prediction: Positive (98%)

13. Challenges & Frontiers

Long Context

Standard self-attention is O(n²) in sequence length. Solutions: sparse attention, linear attention, FlashAttention, and sliding window approaches.

Hallucination

Language models can generate plausible but factually incorrect text. Retrieval-Augmented Generation (RAG) grounds outputs in retrieved documents.

Multilingual NLP

Models like mBERT and XLM-R handle 100+ languages. Zero-shot cross-lingual transfer: train on English, test on other languages.

Efficiency

Large models are expensive. Techniques: knowledge distillation, quantisation (INT8/INT4), pruning, LoRA and QLoRA for parameter-efficient fine-tuning.
