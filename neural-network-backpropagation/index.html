1. What Is a Neural Network?

The Basic Components

A neural network is a computational model inspired by biological neurons. It consists of:

  • Neurons: Basic processing units that receive inputs and produce outputs.
  • Layers: Collections of neurons organized into input, hidden, and output layers.
  • Weights: Learnable parameters that scale the importance of each connection between neurons.
  • Biases: Additional learnable parameters that shift the activation function output.
  • Activation Functions: Non-linear functions (ReLU, Sigmoid, Tanh) that introduce complexity.
Key Insight: A neural network learns by adjusting weights and biases to minimize prediction error. The forward pass computes predictions, and backpropagation calculates gradients to update parameters.

How Neurons Work

Each neuron computes a weighted sum of its inputs plus a bias, then applies an activation function:

neuron_output = activation(Σ(weight_i × input_i) + bias)

The activation function introduces non-linearity, allowing neural networks to model complex patterns. Common choices include:

  • ReLU: f(x) = max(0, x) — Most popular in hidden layers
  • Sigmoid: f(x) = 1 / (1 + e^(-x)) — Binary classification outputs
  • Tanh: f(x) = (e^x - e^(-x)) / (e^x + e^(-x)) — Zero-centered sigmoid

2. The Forward Pass

Computing Predictions

The forward pass propagates input data through the network layer by layer to produce predictions. Each layer's output becomes the next layer's input.

For a simple network with one hidden layer:

z1 = w1·x1 + w2·x2 + b1 a1 = ReLU(z1) z2 = w3·a1 + w4·a2 + b2 output = sigmoid(z2)
Step-by-Step Process:
  1. Input layer receives raw data (features)
  2. First hidden layer: compute z = Wx + b, apply activation
  3. Subsequent layers: repeat with new inputs
  4. Output layer: final predictions (with appropriate activation)

Information Flow

Think of the forward pass as information flowing from left to right:

  • Input: Raw features (x1, x2, ...)
  • Weighted Sum: Linear combination of inputs scaled by weights
  • Activation: Non-linear transformation applied
  • Hidden Layer Output: Becomes input to next layer
  • Output: Final prediction(s)

4. Loss Functions

Mean Squared Error (MSE)

Used primarily for regression tasks. Measures the average squared difference between predicted and actual values.

MSE = (1/n) × Σ(y_true - y_pred)²

Properties: Penalizes large errors heavily; sensitive to outliers; always non-negative.

Cross-Entropy Loss

Used primarily for classification tasks. Measures the difference between predicted probability distributions and true labels.

CrossEntropy = -Σ(y_true × log(y_pred))

Properties: Naturally aligns with probability outputs; well-suited for classification; numerically stable variants exist.

Important: The choice of loss function significantly impacts training. MSE works well for regression; Cross-Entropy works well for classification. The loss function's gradient drives backpropagation.

5. Backpropagation Explained

The Chain Rule Foundation

Backpropagation uses the chain rule from calculus to compute gradients. If we want to know how a weight affects the loss, we chain partial derivatives:

∂Loss/∂w = (∂Loss/∂output) × (∂output/∂z) × (∂z/∂w)

This breaks the complex derivative into manageable pieces that we can compute layer by layer.

Gradient Flow Backwards

Backpropagation flows error signals backwards through the network:

  • Step 1: Compute loss from predictions
  • Step 2: Calculate gradient of loss w.r.t. output
  • Step 3: Propagate gradient backwards through output layer
  • Step 4: Continue for each hidden layer using chain rule
  • Step 5: Calculate gradient for each weight: ∂L/∂w
Key Insight: Once you have ∂Loss/∂weight, you update the weight in the opposite direction: w_new = w_old - learning_rate × ∂Loss/∂w

Weight Updates

After computing gradients, weights are updated using gradient descent:

w_new = w_old - α × (∂L/∂w) b_new = b_old - α × (∂L/∂b)

Where α (alpha) is the learning rate. A larger learning rate means bigger steps, but risks overshooting. A smaller learning rate means slower, more stable learning.

7. The Training Loop

Putting It All Together

Training combines the forward pass and backpropagation in a repeating cycle:

for each epoch: for each batch in training_data: forward_pass(batch) compute_loss(predictions, true_labels) backward_pass() update_weights()

Key Concepts

  • Epoch: One complete pass through the entire training dataset.
  • Batch: A subset of training data processed together (trade-off between memory and gradient stability).
  • Iteration: One gradient update step, equivalent to one batch.
  • Convergence: When the loss stops improving significantly, indicating the network has learned.
  • Overfitting: When the network memorizes training data instead of learning generalizable patterns (detected by validation loss increasing).
Training Strategy: Split data into training, validation, and test sets. Monitor validation loss to detect overfitting early. Use early stopping to halt training when validation loss plateaus.

8. Applications & Tips

Common Challenges & Solutions

Vanishing Gradients

Problem: In deep networks, gradients become exponentially smaller moving backwards through layers, especially with sigmoid/tanh activations. Weights in early layers barely update.

Solutions:

  • Use ReLU activations instead of sigmoid/tanh
  • Use batch normalization to stabilize gradient flow
  • Use skip connections (residual networks)
  • Carefully initialize weights (He initialization for ReLU)

Exploding Gradients

Problem: The opposite of vanishing gradients—gradients become exponentially larger, causing unstable training and NaN losses.

Solutions:

  • Gradient clipping: cap gradient magnitude to a threshold
  • Reduce learning rate
  • Use batch normalization

Batch Normalization

Normalizes layer inputs to have zero mean and unit variance:

z_norm = (z - batch_mean) / sqrt(batch_variance + epsilon) z_scaled = γ × z_norm + β

Benefits: Stabilizes learning, allows higher learning rates, reduces internal covariate shift, provides regularization effect.

Dropout Regularization

Randomly deactivates neurons during training (typically 20-50% dropout rate):

Benefits: Prevents co-adaptation of neurons, reduces overfitting, forces network to learn robust features, effectively creates ensemble of networks.

Note: Dropout is only applied during training; all neurons are active during inference.

Training Tips:
  • Normalize/standardize input features to zero mean and unit variance
  • Initialize weights carefully (avoid all zeros or very large values)
  • Use appropriate activation functions for each layer type
  • Monitor both training and validation loss for overfitting
  • Use learning rate scheduling to decay learning rate over time
  • Consider momentum or adaptive optimizers (Adam, RMSprop) instead of vanilla SGD

Real-World Applications

  • Computer Vision: Image classification, object detection, segmentation (CNNs)
  • Natural Language Processing: Text classification, machine translation, language modeling (RNNs, Transformers)
  • Time Series: Forecasting, anomaly detection (LSTMs, GRUs)
  • Speech: Speech recognition, speaker identification
  • Reinforcement Learning: Game playing, robotics, autonomous control
