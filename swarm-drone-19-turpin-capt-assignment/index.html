🎯 Section 1: Paper Overview

You have 50 drones hovering in the air. You want them to rearrange into a star pattern. Which drone goes where? And how do they get there without colliding? This is the Concurrent Assignment and Planning of Trajectories (CAPT) problem, elegantly solved by Turpin, Michael, and Kumar at UPenn's GRASP Lab.

Key Achievement

Proved that the optimal assignment (minimizing total distance) automatically produces collision-free trajectories when robots move along straight lines. No separate collision avoidance needed!

🧩 Section 2: The Assignment Problem

Given N robots at positions {s₁, s₂, ..., sₙ} and N goal positions {g₁, g₂, ..., gₙ}, find a permutation σ that maps each robot to a goal:

minimize Σᵢ cost(sᵢ, g_σ(i)) over all permutations σ

There are N! possible assignments. For 20 robots, that's 2.4 × 10¹⁸ possibilities — far too many to check exhaustively. The Hungarian algorithm solves this in O(N³) time.

But assignment alone isn't enough. Once you know which drone goes where, the drones need collision-free paths. The brilliant insight of CAPT is that these two problems are not independent:

CAPT Theorem: If all robots are identical and move along straight-line paths at equal speed, then the assignment that minimizes total squared distance is automatically collision-free. No separate collision avoidance is needed!

This is remarkable because it reduces a complex multi-robot planning problem (assignment + collision avoidance) to a single, clean optimization (just assignment).

⚙️ Section 3: The CAPT Algorithm

  1. Input: N current robot positions and N goal positions.
  2. Compute cost matrix: C[i][j] = ||sᵢ - gⱼ||² (squared distance from robot i to goal j).
  3. Solve optimal assignment: Use the Hungarian algorithm to find the permutation σ* that minimizes total cost.
  4. Generate trajectories: Each robot i moves in a straight line from sᵢ to g_σ*(i) at constant speed.
  5. Execute: All robots start simultaneously and arrive simultaneously.
σ* = argmin_σ Σᵢ ||sᵢ - g_σ(i)||²

The key properties:

  • Optimal: Minimizes total travel distance (squared).
  • Collision-free: Guaranteed by the optimality theorem — no paths cross.
  • Simultaneous: All robots start and finish at the same time.
  • Scalable: O(N³) for assignment, O(N) for trajectory generation.

🔢 Section 4: The Hungarian Algorithm

The Hungarian algorithm (Kuhn-Munkres) finds the optimal assignment in O(N³) time. The idea:

StepActionPurpose
1Subtract row minimumsCreate zeros in cost matrix
2Subtract column minimumsMore zeros appear
3Cover all zeros with minimum linesTest if N independent zeros exist
4If N lines needed: done (optimal)Each zero = one assignment
5If < N lines: modify matrixCreate more zeros and repeat
Cost matrix C[i][j] = ||robot_i_pos - goal_j_pos||²

For N = 50 robots, the algorithm performs roughly 50³ = 125,000 operations — trivial for any modern computer. Even for N = 1000, it completes in under a second.

📐 Section 6: Trajectory Generation

Once assignment is computed, trajectories are trivially simple:

pᵢ(t) = sᵢ + t · (g_σ(i) - sᵢ) for t ∈ [0, 1]

Each robot moves along a straight line from start to goal. All robots begin at t=0 and arrive at t=1. The motion is parameterized so all robots travel at speed proportional to their distance — or equivalently, all arrive simultaneously.

Why Straight Lines?

The collision-free guarantee only holds for straight-line trajectories. If robots take curved paths, the optimality theorem breaks down and collisions become possible. However, straight lines are also the shortest paths, so there's no practical reason to curve.

Simultaneous Arrival

All robots must arrive at the same time for the collision-free property to hold. This means robots traveling longer distances move faster. In practice, there's a maximum speed limit, so the total transition time is set by the robot with the longest path:

T_total = max_i(||sᵢ - g_σ(i)||) / v_max

🛡️ Section 7: The Collision-Free Proof

The most elegant result in the paper: optimal assignment implies no collisions. The proof works by contradiction:

  1. Assume the optimal assignment σ* causes two robots i and j to collide at some time t.
  2. At the collision point, robots i and j occupy the same position. Their paths must cross.
  3. If paths cross, we can swap the goal assignments: give robot i the goal of j, and vice versa.
  4. By the triangle inequality, this swap reduces total squared distance.
  5. But σ* was supposed to be optimal — contradiction!
||sᵢ - g_σ(j)||² + ||sⱼ - g_σ(i)||² < ||sᵢ - g_σ(i)||² + ||sⱼ - g_σ(j)||²

If paths cross, swapping is always cheaper. Therefore, the optimal assignment never has crossing paths. Non-crossing straight-line paths in 2D cannot collide. QED.

Intuition: Crossing paths mean two robots travel past each other — wasting distance. The optimal assignment never wastes distance, so paths never cross, so robots never collide. Optimality = Safety.

The greedy algorithm (each robot claims nearest unclaimed goal) is fast but suboptimal: early assignments can force later robots into long detours. CAPT's Hungarian algorithm considers all assignments simultaneously, finding the global optimum.

📈 Section 10: Scaling Analysis

  • N = 10: Hungarian runs in microseconds. Trivial.
  • N = 100: ~1 million operations. Still under 1ms.
  • N = 1,000: ~1 billion operations. Under 1 second.
  • N = 10,000: ~10¹² operations. Minutes — may need optimization.
  • N = 100,000: Impractical with standard Hungarian. Auction algorithms or approximate methods needed.

For practical drone swarms (up to ~1000 drones), CAPT is fast enough for real-time recalculation. Drone show companies like Verge Aero and Intel Shooting Star use similar assignment algorithms for their choreography.

Extensions

  • 3D: The collision-free theorem extends to 3D for straight-line paths.
  • Heterogeneous robots: If robots have different speeds, the cost function must account for arrival time, not just distance.
  • Dynamic reassignment: If goals change mid-flight, recompute the assignment and smoothly transition to new paths.
  • Obstacles: With obstacles, straight lines may not be feasible. CAPT must be combined with path planning.

🌍 Section 13: Applications

Drone Light Shows

The primary commercial application. Intel, Verge Aero, and others use CAPT-style algorithms to choreograph hundreds of drones transitioning between formations in aerial displays.

Warehouse Robotics

Amazon-style warehouse robots (Kiva/Proteus) need collision-free assignments from current positions to pickup/dropoff locations. CAPT provides optimal routing.

Autonomous Parking

Self-driving cars in automated parking garages: each car needs a spot, and the assignment should minimize total movement while avoiding collisions.

Swarm Reconfiguration

Any multi-robot system that transitions between formations: military drone swarms, satellite constellations, or modular self-reconfiguring robots.

🎯 Section 14: Key Takeaways

  1. Optimality implies safety: The optimal assignment (minimum total squared distance) automatically produces collision-free straight-line paths. This is the paper's central theorem.
  2. The Hungarian algorithm solves assignment in O(N³): Fast enough for real-time use with up to ~1000 robots.
  3. Crossing paths are suboptimal: If two paths cross, swapping the assignments reduces total cost. Therefore, optimal assignments never cross.
  4. Simultaneous start/finish is required: All robots must begin and end their transitions at the same time for the collision-free guarantee.
  5. Straight-line paths are key: The theorem requires straight-line trajectories. Curved paths break the guarantee.
  6. This is the math behind drone shows: Commercial drone show companies use CAPT-style algorithms for their choreography software.
  7. Elegant reduction: CAPT reduces a hard problem (multi-robot planning with collision avoidance) to a solved problem (optimal assignment). This kind of reduction is the gold standard of algorithmic thinking.
