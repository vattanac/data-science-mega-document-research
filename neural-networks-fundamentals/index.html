1. Biological Inspiration

Artificial neural networks are loosely inspired by the brain. Biological neurons receive signals through dendrites, process them in the cell body, and transmit output through axons. Similarly, artificial neurons receive weighted inputs, apply an activation function, and produce an output.

Think of a neuron as a tiny decision-maker. It listens to many inputs (friends giving advice), weights each one by how much it trusts that friend, adds them up, and makes a yes/no decision if the total "convincing power" crosses a threshold.

Key differences from biology: artificial neurons operate on precise numbers, use continuous (not spiking) signals, and are arranged in clean layers rather than the brain's complex web. Still, this simple model is powerful enough to learn almost any function.

2. The Perceptron

The perceptron (Rosenblatt, 1958) is the simplest neural network — a single neuron that computes a weighted sum of inputs plus a bias, then applies a step function:

z = w₁x₁ + w₂x₂ + ... + wₙxₙ + b = Σ wᵢxᵢ + b
output = 1 if z ≥ 0, else 0

The perceptron can learn any linearly separable pattern — it draws a hyperplane to divide two classes. It cannot solve XOR or any non-linearly separable problem (this was the famous Minsky & Papert criticism in 1969).

Perceptron Learning Rule

wᵢ ← wᵢ + α · (y_true − y_pred) · xᵢ

If the prediction is correct, no update. If wrong, weights are nudged toward the correct answer. The learning rate α controls step size. This is guaranteed to converge for linearly separable data.

3. Activation Functions

Activation functions introduce nonlinearity — without them, stacking layers would just compute another linear function. They determine what a neuron "fires."

4. Multi-Layer Networks (MLPs)

A Multi-Layer Perceptron (MLP) stacks multiple layers of neurons. The magic: by adding hidden layers with nonlinear activations, the network can approximate any continuous function (Universal Approximation Theorem).

Network Structure

  • Input layer: one neuron per feature (no computation, just passes data through)
  • Hidden layers: where the learning happens — each neuron transforms its inputs nonlinearly
  • Output layer: produces the final prediction (1 neuron for regression/binary, C neurons for C classes)

Parameters: a network with layers of size [n₀, n₁, n₂, ..., nₖ] has Σ(nᵢ × nᵢ₊₁ + nᵢ₊₁) total parameters (weights + biases).

An MLP is like a factory assembly line. Raw materials (input) pass through multiple processing stations (hidden layers). Each station transforms the product. Early stations do basic processing (detect edges), later stations do sophisticated work (recognize faces). The final station packages the output.

5. Forward Propagation

Forward propagation computes the network's output from input to output, layer by layer:

Layer ℓ: z⁽ℓ⁾ = W⁽ℓ⁾ · a⁽ℓ⁻¹⁾ + b⁽ℓ⁾, a⁽ℓ⁾ = f(z⁽ℓ⁾)

Where W⁽ℓ⁾ is the weight matrix, a⁽ℓ⁻¹⁾ is the previous layer's activation, b⁽ℓ⁾ is the bias vector, and f is the activation function. The input layer activation is simply a⁽⁰⁾ = x.

This is a composition of functions: ŷ = f₃(W₃ · f₂(W₂ · f₁(W₁ · x + b₁) + b₂) + b₃). The depth (number of compositions) is what makes "deep" learning powerful.

6. Loss Functions

The loss function measures how wrong the network's predictions are. Training = minimizing loss.

TaskLossFormula
RegressionMean Squared ErrorL = (1/n) Σ(yᵢ − ŷᵢ)²
Binary ClassificationBinary Cross-EntropyL = −(1/n) Σ[yᵢ log ŷᵢ + (1−yᵢ) log(1−ŷᵢ)]
Multi-classCategorical Cross-EntropyL = −(1/n) Σ Σ yᵢⱼ log ŷᵢⱼ
Cross-entropy penalizes confident wrong predictions very heavily (log of near-zero). This is why it converges faster than MSE for classification.

7. Backpropagation

Backpropagation is the algorithm that computes how to update each weight. It applies the chain rule backwards through the network to find ∂L/∂w for every weight.

The Chain Rule in Action

∂L/∂W⁽ℓ⁾ = ∂L/∂a⁽ℓ⁾ · ∂a⁽ℓ⁾/∂z⁽ℓ⁾ · ∂z⁽ℓ⁾/∂W⁽ℓ⁾
  1. Compute loss: compare ŷ to y using the loss function.
  2. Output gradient: ∂L/∂a⁽ᴸ⁾ (how does loss change with output?).
  3. Propagate backward: at each layer, compute δ⁽ℓ⁾ = ∂L/∂z⁽ℓ⁾ using the layer above's delta.
  4. Compute weight gradients: ∂L/∂W⁽ℓ⁾ = δ⁽ℓ⁾ · a⁽ℓ⁻¹⁾ᵀ.
  5. Update weights: W⁽ℓ⁾ ← W⁽ℓ⁾ − α · ∂L/∂W⁽ℓ⁾.
Backpropagation is like blame assignment. The boss (loss) is unhappy with the result. They figure out which department (output layer) contributed most to the problem, that department traces it to sub-departments (hidden layers), all the way back to individual workers (weights). Everyone gets feedback proportional to their contribution to the error.

Vanishing & Exploding Gradients

Vanishing: If activation derivatives are small (sigmoid/tanh), gradients shrink exponentially through layers. Early layers barely learn. Fix: use ReLU, batch normalization, residual connections.

Exploding: If weights are large, gradients grow exponentially. Training becomes unstable. Fix: gradient clipping, proper initialization (He or Xavier).

8. Training Process

Training Loop

  1. Initialize weights (He init for ReLU, Xavier for tanh/sigmoid).
  2. Forward pass: compute predictions for a mini-batch.
  3. Compute loss: measure prediction error.
  4. Backward pass: compute gradients via backpropagation.
  5. Update weights: optimizer step (SGD, Adam, etc.).
  6. Repeat for all mini-batches = 1 epoch. Train for many epochs.

Weight Initialization

MethodFormulaUse With
Xavier/GlorotW ~ N(0, 2/(nᵢₙ + nₒᵤₜ))Sigmoid, Tanh
HeW ~ N(0, 2/nᵢₙ)ReLU, Leaky ReLU

Good initialization keeps activations and gradients in a healthy range across layers, preventing vanishing/exploding at the start of training.

9. Architecture Design

How Many Layers?

  • 1 hidden layer: can approximate any function (theoretically), good for simple tasks.
  • 2–3 hidden layers: good default for most tabular data problems.
  • Deep (5+ layers): needed for complex patterns (images, language). Requires careful design (batch norm, residual connections, dropout).

How Many Neurons Per Layer?

  • Common pattern: funnel shape — wider early layers, narrowing toward output.
  • Start with layer sizes between input and output dimension.
  • Powers of 2 (64, 128, 256) are common (GPU efficiency).
  • More neurons = more capacity but more overfitting risk.

10. Regularization

Neural networks easily overfit (memorize training data). Regularization keeps them generalizing:

  • L2 Regularization (Weight Decay): add λ·Σw² to loss. Penalizes large weights, encourages small distributed weights.
  • Dropout: randomly set p% of neurons to 0 during training. Forces redundancy — the network can't rely on any single neuron. Typically p = 0.2–0.5.
  • Batch Normalization: normalize layer inputs to zero mean, unit variance. Stabilizes training and acts as mild regularization.
  • Early Stopping: stop training when validation loss starts increasing. Simple and effective.
  • Data Augmentation: artificially expand training set (rotate, flip, crop images). More data = less overfitting.
In practice, use a combination: batch normalization + dropout + early stopping is a strong default. Add L2 regularization if still overfitting.

14. Summary & Key Takeaways

  1. Neurons compute weighted sums + nonlinear activations. The nonlinearity is crucial.
  2. Stacking layers creates universal function approximators. Depth enables hierarchical feature learning.
  3. Backpropagation efficiently computes gradients via the chain rule, making training possible.
  4. ReLU is the default activation. Use He initialization. Use Adam optimizer as default.
  5. Regularization (dropout, batch norm, early stopping) prevents overfitting.
  6. Architecture depends on data: MLPs for tabular, CNNs for images, Transformers for sequences.
  7. Start simple (1–2 hidden layers) and add complexity only if needed.
