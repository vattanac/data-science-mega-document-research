1. Types of Missing Data

Missing Completely At Random (MCAR)

Missing data that has no relationship to any observed or unobserved variables. The probability of a value being missing is independent of both the variable itself and all other variables.

Example: A sensor randomly fails during measurement, regardless of the actual value or other conditions. This is the "best case scenario" for missing data handling.
P(missing | X) = P(missing)
The missingness is completely random and non-informative.

Implications: Any analysis method that ignores the missing data is valid, though power is reduced. Listwise or pairwise deletion is acceptable.

Missing At Random (MAR)

Missing data depends on observed variables, but not on the unobserved values themselves. Given the observed data, the mechanism is random.

Example: Younger patients are less likely to report cholesterol levels (but this is captured by age in your data). The missingness depends on age (observed), not on cholesterol itself (unobserved).
P(missing | X, Y) = P(missing | X)
Missingness depends on observed data X, not on Y itself.

Implications: Deletion is problematic and introduces bias. Imputation methods like MICE (Multiple Imputation by Chained Equations) are appropriate. Multiple imputation is preferred.

Missing Not At Random (MNAR)

Missing data depends on the unobserved values themselves. The probability of missingness is related to the missing value itself.

Example: People with very high or very low income are more likely to skip income questions. Income is missing because of the income itself.
P(missing | Y) ≠ P(missing)
Missingness depends directly on the unobserved value Y.

Implications: This is the most challenging scenario. Deletion introduces severe bias. Imputation requires careful assumptions or sensitivity analysis. Often requires domain knowledge or auxiliary information.

2. Why Missing Data Matters

Statistical Bias

Depending on how you handle missing data, you can introduce systematic errors into your analysis:

  • Selection Bias: If data is missing due to the values themselves (MNAR), ignoring missingness leads to biased estimates.
  • Underestimation of Variance: Listwise deletion reduces variance estimates because you're working with a subset of data.
  • False Associations: Simple deletion can create spurious correlations between variables.

Reduced Statistical Power

Missing data reduces the effective sample size, which decreases your ability to detect real effects:

Effective Sample Size = N × (1 - percentage missing)
10% missing data → ~10% reduction in power
  • Wider confidence intervals
  • Higher p-values for hypothesis tests
  • Inability to detect important effects

Invalid Conclusions

Poor handling of missing data can lead to fundamentally wrong conclusions:

  • Incorrect parameter estimates
  • Unreliable statistical tests
  • Misleading predictive models
  • Inability to compare groups fairly (if missingness differs by group)

Key Takeaway

How you handle missing data is not optional—it's a fundamental part of your analysis that directly affects the validity of your results. Transparency and careful consideration of the missingness mechanism is essential.

3. Strategy 1: Deletion

Listwise Deletion (Complete Case Analysis)

Remove any observation (row) that has missing data in any variable. Only analyze cases with complete data.

Keep only observations where all variables are non-missing
Effective n = number of complete cases

Pros:

  • Simple to implement (one line of code)
  • No assumptions required
  • Analysis is straightforward using standard methods

Cons:

  • Severe sample size reduction (especially with many variables)
  • Introduces bias if data is MAR or MNAR
  • Wastes information from partially complete observations
  • Assumption of MCAR is rarely justified
If you have 10 variables and 5% missing in each, listwise deletion could remove ~41% of your data!

Pairwise Deletion

Use all available data for each analysis. For correlations/associations, use all pairs of observations where both variables are observed.

For each pair of variables: use observations where both are non-missing
Sample size may vary by variable pair

Pros:

  • Retains more data than listwise deletion
  • Better statistical power
  • Simple to implement

Cons:

  • Different analyses use different sample sizes (confusing interpretation)
  • Can produce non-positive definite correlation matrices
  • Still biased if data is MAR or MNAR
  • Standard errors may be inaccurate
When to Use Deletion: Only when data is truly MCAR AND the amount of missing data is small (<5%). Even then, consider imputation first.

4. Strategy 2: Imputation

Mean Imputation

Replace missing values with the mean of non-missing values for that variable.

imputed_value = mean(non-missing values)
Example: Missing age = 42.5 (average of observed ages)

Pros:

  • Preserves the mean of the variable
  • Simple and fast
  • No assumptions required

Cons:

  • Reduces variance - artificially concentrates values around the mean
  • Underestimates standard errors and confidence intervals
  • Weakens correlations with other variables
  • Not suitable for categorical data
Mean imputation is often not recommended except for exploratory analysis. It violates statistical assumptions.

Median Imputation

Replace missing values with the median of non-missing values. More robust to outliers than mean.

Pros:

  • Robust to outliers
  • Preserves the median
  • Better for skewed distributions

Cons:

  • Still reduces variance like mean imputation
  • Same statistical issues apply
  • Not suitable for categorical data

Mode Imputation

Replace missing values with the most frequently occurring value (category). Useful for categorical variables.

Pros:

  • Appropriate for categorical data
  • Preserves the mode
  • Simple implementation

Cons:

  • Reduces variance significantly
  • Can amplify the most common category artificially
  • Only preserves frequencies, not relationships

Forward Fill & Backward Fill

Primarily for time series data. Forward fill uses the previous observation; backward fill uses the next observation.

Forward Fill: impute with last observed value
Backward Fill: impute with next observed value

Use Cases:

  • Time series with continuous phenomena (temperature, stock prices)
  • When values are expected to change slowly

Limitations:

  • Only appropriate for ordered/sequential data
  • Can introduce artifacts if gaps are large
General Rule: Simple imputation methods (mean, median, mode) preserve the variable's distribution poorly and underestimate uncertainty. Use them only for preliminary exploration. For serious analysis, consider advanced methods like MICE or KNN.

5. Strategy 3: Advanced Imputation Methods

K-Nearest Neighbors (KNN) Imputation

For each observation with missing data, find the K most similar observations (based on non-missing values) and impute using their average or weighted average.

imputed_value = mean(values in K nearest neighbors)
Often K=5 or K=10; uses distance metric (Euclidean, Manhattan)

Pros:

  • Considers relationships between variables
  • Preserves data distribution better than simple methods
  • Can handle both numerical and categorical data
  • Relatively simple to understand and implement

Cons:

  • Computationally expensive for large datasets
  • Requires choosing K (often requires cross-validation)
  • Still underestimates uncertainty
  • Sensitive to outliers and scale of variables

Regression Imputation

Use other variables to predict missing values. Build a regression model from complete cases and predict missing values.

Y_missing = β₀ + β₁X₁ + β₂X₂ + ... (fitted from complete cases)
Stochastic variant adds random error: Y_missing = predicted + ε

Pros:

  • Uses information from other variables
  • Deterministic version preserves variable mean
  • Stochastic version better preserves variance
  • Flexible (linear, logistic, tree-based models)

Cons:

  • Assumes specific relationship between variables
  • Can be biased if model is misspecified
  • Deterministic version underestimates uncertainty
  • More complex to implement correctly
Tip: Always use the stochastic version (add residual noise) to avoid artificially reducing variance.

Multiple Imputation by Chained Equations (MICE)

The gold standard for handling missing data. Iteratively impute missing values using conditional distributions, creating multiple complete datasets. Analyze each dataset separately and combine results.

1. Initialize missing values (e.g., mean imputation)
2. For each variable with missing data:
  Fit model predicting it from other variables
  Impute missing values from predictive distribution
3. Repeat steps 2 for multiple iterations
4. Create m complete datasets (typically m=5-20)
5. Analyze each dataset, combine results using Rubin's rules

Pros:

  • Preserves relationships between variables
  • Properly quantifies uncertainty with multiple imputations
  • Flexible: can use different models for different variables
  • Valid under MAR assumption (not MCAR)
  • Produces valid standard errors and confidence intervals
  • Gold standard in statistical practice

Cons:

  • More complex to implement and understand
  • Computationally intensive
  • Results depend on model specification
  • Requires careful combination of results (Rubin's rules)
When to Use: When data is MAR, when missing data is not minimal (>5%), and when statistical inference is important. This is the preferred method in most modern applications.

9. Best Practices

1. Investigate Missing Patterns First

Always start with exploration, not deletion.

  • Create a missing data visualization (heatmap showing which values are missing)
  • Calculate the percentage of missing data per variable
  • Look for patterns: Are certain variables always missing together?
  • Check if missingness correlates with other variables
  • This helps you determine if data is MCAR, MAR, or MNAR
Tools: Use df.isnull().sum() in Python/pandas, or visualize with heatmaps to see patterns.

2. Never Just Delete

Listwise deletion should be your last resort, not first choice.

  • Even small amounts of missing data can cause large losses: 10 variables with 5% missing each = ~40% data loss
  • If you do delete, justify it: prove data is MCAR and loss is minimal
  • Always compare results with/without deletion (sensitivity analysis)
Deletion is only defensible when missing data is trivial (<5%) AND you can justify MCAR.

3. Choose the Right Method for Your Situation

  • Simple imputation (mean/median): Use only for preliminary exploration, never for final analysis
  • Advanced imputation (MICE, KNN): Use for serious analysis where you need valid inference
  • Time series: Consider forward/backward fill for sequential data
  • Categorical data: Use mode or classification-based imputation, not mean

4. Document Your Decisions

Transparency is critical. Document:

  • How much data was missing (by variable and overall)
  • Whether you investigated the missing pattern (MCAR/MAR/MNAR assessment)
  • Which imputation method you used and why
  • Any parameters (e.g., K=5 for KNN, m=20 for MICE iterations)
  • How results changed with different methods (sensitivity analysis)
Include a table in your report showing "% missing by variable" and state your assumptions about the missing mechanism.

5. Validate Your Imputation

  • Sanity checks: Are imputed values realistic? Do they fall within expected ranges?
  • Distribution checks: After imputation, does the distribution still look reasonable?
  • Sensitivity analysis: Run your analysis with different imputation methods. Do conclusions change?
  • Multiple imputation: Use MICE to get uncertainty estimates (not just point estimates)

6. Use Multiple Imputation for Inference

For statistical tests, confidence intervals, or p-values, multiple imputation (MICE) is the standard:

  • Creates multiple complete datasets reflecting uncertainty about missing values
  • Analyze each dataset and combine results using Rubin's rules
  • Produces valid standard errors and confidence intervals
  • Properly accounts for uncertainty due to missingness
Simple imputation methods cannot produce valid standard errors for inference.

7. Consider Auxiliary Information

  • Include all relevant variables in your imputation model, even if not needed for final analysis
  • More variables = better imputation (under MAR assumption)
  • For MNAR, use external/domain knowledge to validate assumptions

Checklist Before Publishing/Presenting

  • ☐ Assessed missing mechanism (MCAR/MAR/MNAR)
  • ☐ Justified imputation method choice
  • ☐ Reported amount of missing data
  • ☐ Documented all parameters (K, iterations, models used)
  • ☐ Validated imputed values (sanity checks)
  • ☐ Conducted sensitivity analysis (different methods compared)
  • ☐ Used appropriate method for inference (MICE for p-values, CI)
  • ☐ Acknowledged limitations of missing data handling
