1. Paper Overview

Title: On-board Communication-based Relative Localization for Collision Avoidance in Micro Air Vehicle Swarms

Authors: Mario Coppola, Kimberly N. McGuire, Kirk Y.W. Scheper, Guido C.H.E. de Croon

Published: Autonomous Robots, Vol. 42, pp. 1787–1805 (2018)

Institution: TU Delft — Micro Air Vehicle Laboratory (MAVLab)

This paper presents a method for MAVs to estimate the relative positions of nearby swarm members using only Received Signal Strength Indicator (RSSI) from Bluetooth communication — no cameras, no GPS, no external infrastructure. Each drone measures the signal strength of messages from neighbors and combines this with its own velocity estimate to track neighbors via an Extended Kalman Filter (EKF).

Core Contribution: A lightweight, infrastructure-free relative localization method that enables collision avoidance in MAV swarms using only the wireless communication hardware already present on every drone — no additional sensors required.

The key advantage of RSSI-based localization: zero additional hardware cost. Every drone already has a Bluetooth or WiFi radio for communication. RSSI is a free byproduct of receiving any wireless message. The challenge is that RSSI is extremely noisy — can the signal be made useful?

3. RSSI-Based Range Estimation

When a radio signal travels through air, its strength decreases with distance following the log-distance path loss model:

RSSI(d) = RSSI(d₀) - 10·n·log₁₀(d/d₀) + X_σ

where d is distance, d₀ is a reference distance (1m), n is the path loss exponent (2.0 in free space, 2-4 indoors), and X_σ is zero-mean Gaussian noise (σ = 4-8 dB typically).

The Challenge: Noise

RSSI measurements are notoriously noisy due to multipath reflections, body shadowing, antenna orientation, and environmental changes. A single RSSI reading might give a range estimate anywhere from 50% to 200% of the true distance. This is why most systems avoid RSSI for localization.

The Solution: Temporal Filtering

While individual RSSI readings are noisy, their average over time converges. The paper uses an Extended Kalman Filter (EKF) that fuses many noisy RSSI measurements over time with the drone's own velocity estimate, producing a much more accurate relative position estimate.

4. Relative Localization Pipeline

  1. Message Exchange: Each drone broadcasts its velocity estimate (from onboard IMU + optical flow) via Bluetooth at 10 Hz.
  2. RSSI Measurement: When a message is received, the radio chip reports the RSSI value (free — no extra hardware).
  3. Range Estimate: RSSI is converted to a noisy distance estimate using the calibrated path-loss model.
  4. EKF Update: The range estimate is fused with the drone's own velocity and the neighbor's broadcast velocity in an EKF that tracks the relative position (x, y) of each neighbor.
  5. Collision Avoidance: If a neighbor's estimated relative position is within a safety radius, a repulsive velocity command is generated.
// Per neighbor, per message received:
rssi = receive_message(neighbor_id)
d_raw = rssi_to_distance(rssi)         // Noisy!
v_self = get_own_velocity()             // From optical flow
v_neighbor = message.velocity           // Broadcast by neighbor

// EKF prediction step
x_pred = x_prev + (v_neighbor - v_self) * dt

// EKF update step with range measurement
K = P_pred * H' / (H * P_pred * H' + R)
x_est = x_pred + K * (d_raw - ||x_pred||)
P_est = (I - K*H) * P_pred

5. Extended Kalman Filter Details

State Vector

For each tracked neighbor, the EKF maintains a 2D relative position state:

x = [Δx, Δy]ᵀ (relative position of neighbor)

Process Model

The prediction uses the velocity difference between the two drones:

x(k+1) = x(k) + (v_neighbor - v_self) · Δt + w

Measurement Model

The measurement is the estimated range from RSSI:

z = √(Δx² + Δy²) + v (range = distance + noise)

The EKF handles the nonlinearity of the range measurement through linearization. The measurement noise covariance R is set large (reflecting RSSI noise), while the process noise Q is set based on velocity estimation accuracy.

Observability

Range-only measurements from a single sensor are not fully observable for 2D position (you know the distance but not the direction). The key insight is that relative motion between the drones creates observability — as the drones move, the changing range disambiguates the direction. This is why velocity information from both drones is critical.

7. Collision Avoidance Strategy

Once relative positions are estimated, collision avoidance uses a simple but effective repulsive potential field:

v_avoid = K_rep · (r_safe - d_est) / d_est · n̂ when d_est < r_safe

where d_est is the estimated distance to the neighbor, r_safe is the safety radius, and n̂ is the unit vector pointing away from the neighbor.

Design Choices

  • Conservative safety radius: Set larger than needed to account for RSSI estimation errors (~1m safety for ~0.5m accuracy).
  • Priority override: Collision avoidance always overrides mission commands — safety first.
  • Symmetric avoidance: Both drones see each other and both steer away, doubling the effective separation speed.
  • Graceful degradation: If RSSI estimates are very uncertain, the safety radius expands automatically (uncertainty-aware avoidance).

8. Swarm Behaviors Enabled

Beyond collision avoidance, the relative localization enables several collective behaviors:

Dispersion

Drones spread out to cover maximum area while maintaining communication range. Each drone moves away from its nearest neighbor.

Aggregation

Drones converge toward each other to form a cluster. Useful for rendezvous after separate tasks.

Flocking

Combined separation + alignment + cohesion using RSSI-derived relative positions. No vision needed.

Area Coverage

Cooperative exploration where drones repel each other while moving forward, naturally spreading across the search area.

10. Experimental Validation

Platform: Crazyflie 2.0 Nano-Quadrotor

FeatureSpec
Size92mm motor-to-motor
Weight27g
RadioBluetooth Low Energy (BLE) + nRF24 2.4 GHz
SensorsIMU, barometer, optical flow (PMW3901)
ProcessorSTM32F4 (168 MHz ARM Cortex-M4)

Results

  • Range accuracy: ±0.5m at distances up to 3m (after EKF filtering)
  • Bearing estimation: ±30° when drones are moving (range-only, no direct bearing)
  • Collision avoidance success: 100% — zero collisions across all flight tests
  • Tested with: Up to 4 Crazyflie drones flying simultaneously indoors
  • Computation: EKF runs entirely onboard the Crazyflie's 168 MHz processor
  • No external infrastructure: No motion capture, no beacons, no GPS

14. Key Takeaways

  1. Zero-cost sensing: RSSI is a free byproduct of wireless communication. No additional hardware means no additional weight, cost, or power draw.
  2. Good enough for safety: ±0.5m accuracy isn't great for formation flying, but it's sufficient for collision avoidance with conservative safety margins.
  3. EKF is the key: Individual RSSI readings are useless. Temporal filtering with velocity information makes them useful.
  4. Motion creates observability: Range-only measurements become informative when drones are moving, because changing range disambiguates direction.
  5. Fully onboard: Everything runs on a 168 MHz microcontroller. No ground station, no cloud, no external compute.
  6. Enabling technology: This makes swarm experiments accessible to any lab with basic hardware — democratizing swarm research.
  7. Graceful degradation: When RSSI is very noisy, the safety radius automatically expands rather than failing catastrophically.
Impact: Coppola et al. showed that even the noisiest sensor — RSSI — can enable safe swarm flight when combined with intelligent filtering. This work makes swarm robotics accessible to researchers without expensive infrastructure, and paves the way for truly deployable nano-drone swarms.
