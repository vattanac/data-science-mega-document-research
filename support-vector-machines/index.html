Support Vector Machines

Maximum-margin classifiers, the kernel trick & beyond

Machine Learning Classification Kernels Margin Hyperplanes

1. The Big Idea

A Support Vector Machine finds the hyperplane that separates two classes with the maximum margin — the widest possible gap between the closest points of each class. Those closest points are the support vectors.

Why maximum margin? A wider margin means better generalisation — the classifier is less sensitive to small perturbations in the data. This is one of the most elegant ideas in machine learning.

SVMs are effective in high-dimensional spaces, work well with clear margins of separation, and remain robust when the number of features exceeds the number of samples.

2. Linear SVM — Hard Margin

For linearly separable data, we seek a hyperplane w·x + b = 0 such that:

y_i (w · x_i + b) ≥ 1 for all i

The margin width is 2/‖w‖. Maximising the margin is equivalent to minimising ‖w‖²:

minimise ½ ‖w‖² subject to y_i (w · x_i + b) ≥ 1 ∀i

This is a convex quadratic program. The solution depends only on the support vectors — all other points can be removed without changing the decision boundary.

3. Soft Margin — Handling Noise

Real data is rarely perfectly separable. The soft margin SVM introduces slack variables ξ_i to allow some misclassifications:

minimise ½ ‖w‖² + C Σ ξ_i subject to y_i (w · x_i + b) ≥ 1 − ξ_i, ξ_i ≥ 0
  • C is large: Small margin, fewer misclassifications (risk of overfitting)
  • C is small: Wide margin, more misclassifications allowed (risk of underfitting)

C controls the bias-variance trade-off. It's the most important hyperparameter for a linear SVM.

5. The Dual Formulation

Using Lagrange multipliers α_i, the SVM can be reformulated as the dual problem:

maximise Σ α_i − ½ ΣΣ α_i α_j y_i y_j (x_i · x_j) subject to 0 ≤ α_i ≤ C, Σ α_i y_i = 0

Key insight: the data only appears as dot products x_i · x_j. This is what enables the kernel trick.

Points with α_i > 0 are the support vectors. Points with α_i = C are on the wrong side of the margin (misclassified or within the margin).

6. The Kernel Trick

What if data isn't linearly separable? Map it to a higher-dimensional space where it is! A kernel function K(x_i, x_j) = φ(x_i) · φ(x_j) computes the dot product in the high-dimensional space without ever computing the mapping φ explicitly.

KernelK(x, z)When to Use
Linearx · zLinearly separable data, high-dimensional text
Polynomial(γ x·z + r)^dFeature interactions up to degree d
RBF (Gaussian)exp(−γ ‖x−z‖²)Default choice; handles most nonlinear patterns
Sigmoidtanh(γ x·z + r)Neural-network-like behaviour (rarely used)

The RBF kernel maps data to an infinite-dimensional space. The parameter γ controls the "reach" of each training point — high γ means tight, complex boundaries.

8. Hyperparameter Tuning

C — Regularisation

Controls the penalty for misclassification. Large C = hard margin (low bias, high variance). Small C = soft margin (high bias, low variance). Typical search: [0.01, 0.1, 1, 10, 100].

γ (gamma) — RBF Kernel Width

Controls how far the influence of a single training point reaches. Large γ = tight, complex boundaries (overfitting risk). Small γ = smooth, simple boundaries (underfitting risk). Often search: [0.001, 0.01, 0.1, 1, 10].

Degree (d) — Polynomial Kernel

The degree of the polynomial. Higher = more flexible but slower and prone to overfitting. Common values: 2, 3, 4.

Use grid search with cross-validation to find optimal (C, γ) pairs. Scikit-learn's GridSearchCV automates this.

9. Multi-Class SVM

SVMs are inherently binary classifiers. Two strategies extend them to K classes:

One-vs-Rest (OvR)

Train K binary classifiers, each separating one class from all others. Assign the class whose classifier gives the highest confidence. Requires K classifiers.

One-vs-One (OvO)

Train K(K−1)/2 classifiers, one for each pair of classes. Use majority voting. More classifiers but each trains on a smaller subset.

Scikit-learn uses OvO by default for SVC (faster for large K) and OvR for LinearSVC.

10. Support Vector Regression (SVR)

SVMs can also perform regression. SVR fits a tube of width ε around the data and only penalises points outside the tube:

minimise ½ ‖w‖² + C Σ (ξ_i + ξ*_i) subject to y_i − (w·x_i + b) ≤ ε + ξ_i (w·x_i + b) − y_i ≤ ε + ξ*_i ξ_i, ξ*_i ≥ 0

The ε parameter defines an "insensitivity zone" — errors within this zone are ignored. Points outside become the support vectors.

12. SVM — Strengths & Limitations

StrengthsLimitations
Effective in high-dimensional spacesSlow on large datasets (O(n²) to O(n³) training)
Memory-efficient (only stores support vectors)Sensitive to feature scaling
Robust with proper regularisationNo direct probability estimates (need Platt scaling)
Kernel trick handles nonlinearity elegantlyChoice of kernel & hyperparameters is non-trivial
Strong theoretical guarantees (margin theory)Not ideal for very noisy data with overlapping classes

13. Practical Tips

  • Always scale features — SVMs are sensitive to feature magnitudes. Use StandardScaler or MinMaxScaler.
  • Start with RBF kernel — it's the most versatile. Try linear only if you have very high-dimensional data (text, genomics).
  • Use grid search — search C and γ in log-scale (e.g., 10⁻³ to 10³).
  • For large datasets — consider LinearSVC (uses liblinear, scales better) or SGDClassifier with hinge loss.
  • For probability estimates — set probability=True in SVC (adds Platt scaling, slower).
  • Imbalanced classes — use class_weight='balanced' to adjust C per class.
