What is regression?

Regression is a type of supervised machine learning where the goal is to predict a numeric value from input features.

Example: We want to predict how many ice creams are sold on a given day, based on the temperature.

The model learns a pattern from historical data (days where we know both the temperature and the actual ice cream sales). Once trained, it can predict sales for any new temperature.

Think of it this way: the model finds a mathematical formula that converts temperature into a predicted ice cream count. The process of finding that formula is called training.

Why split? If we test the model on the same data it trained on, it looks perfect — but it hasn't proven it can generalize to new data. The validation set simulates real-world unseen data.

Step 2 — Train the model

We plot the 6 training points on a chart (temperature on x-axis, sales on y-axis). The linear regression algorithm draws the best-fit straight line that minimizes the average distance to all points.

The algorithm found that the best line for our data has the formula:

f(x) = x − 50

The line crosses x=50 at y=0, and rises by 1 ice cream for every 1-degree increase in temperature.

Quick example: Temperature = 77°F → predicted sales = 77 − 50 = 27 ice creams

The dashed red lines show the error — how far off each prediction was. Next, we'll turn all these errors into a single evaluation number.

Metric 1 — Mean Absolute Error (MAE)

The errors are: −2, −3, +3, −1, −2, +3

We don't care about direction — only how large the error is. So we take the absolute value (remove the sign):

MAE result
2.33
In plain English
On average, our predictions were off by 2.33 ice creams

When to use MAE: It's simple and easy to understand. It treats all errors equally — whether you miss by 1 or miss by 5, they contribute proportionally to the score.

Metrics 2 & 3 — MSE and RMSE

The problem with MAE: It treats a small error (1) and a large error (5) proportionally. But large errors are often much worse in practice — being off by 10 is more than 2× as bad as being off by 5.

Mean Squared Error (MSE) fixes this by squaring each error — making larger errors count much more heavily:

6.0
In "squared ice creams" — hard to interpret on its own
RMSE
2.45

Metric 4 — R² (R-Squared)

MAE/MSE/RMSE measure absolute error size. R² asks a different question: what proportion of the variation in sales does our model actually explain?

R² = 0
Model explains nothing. Might as well always guess the average.
R² = 1
Perfect model. Explains 100% of the variation in y.

Our R² = 0.95 means temperature explains 95% of the variation in ice cream sales. Only 5% is left unexplained — random factors like local events, unusual weather, etc.

Step 8 — Iterate & improve

One training run is rarely the end. In practice, data scientists repeat the whole process with different settings until the evaluation metrics are satisfactory.
