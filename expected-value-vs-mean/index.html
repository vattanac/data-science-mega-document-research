📊 Data Science Fundamentals

Expected Value vs Mean

Two powerful concepts that look similar — but answer completely different questions. Master both to think like a data scientist.

E[X] = Σ xᵢ · P(xᵢ) x̄ = Σxᵢ / n Law of Large Numbers Probability Theory
Section 01
The Core Difference
Both produce a single summary number — but they work with fundamentally different things.
MEAN / AVERAGE
x̄

Summarizes data you already collected from the real world.

x̄ = (x₁ + x₂ + ... + xₙ) / n
  • Works on observed, real data
  • All values treated equally
  • No probability needed
  • Describes the past
EXPECTED VALUE
E[X]

Predicts the average outcome before collecting data, using probabilities.

E[X] = Σ xᵢ · P(xᵢ)
  • Works on probabilities
  • Each value weighted by chance
  • Used for decisions & risk
  • Predicts the future
💡 Simple Intuition: Mean looks backward at what happened. Expected Value looks forward at what will likely happen. When you have infinite data, the mean converges to the expected value — this is the Law of Large Numbers.
Section 02 — Step by Step
Example 1: Rolling a Die 🎲
A classic example that shows both concepts clearly and side by side.
Step 1 — Compute the Expected Value

Each face has probability 1/6 ≈ 0.167. Multiply each outcome by its probability, then sum.

⚀P = 1/61×1/6 = 0.17
⚁P = 1/62×1/6 = 0.33
⚂P = 1/63×1/6 = 0.50
⚃P = 1/64×1/6 = 0.67
⚄P = 1/65×1/6 = 0.83
⚅P = 1/66×1/6 = 1.00
E[X] = 1×(1/6) + 2×(1/6) + 3×(1/6) + 4×(1/6) + 5×(1/6) + 6×(1/6)
      = (1 + 2 + 3 + 4 + 5 + 6) / 6
      = 21 / 6 = 3.5
You will never roll a 3.5 — but if you roll thousands of times, the average will be very close to 3.5.

Section 03 — Real World
Example 2: Insurance Company 🏥
Insurance companies use Expected Value every day to set premiums and assess risk.
Scenario: Car Insurance Annual Payout

The insurer doesn't know which individual will have an accident — but they know the probabilities.

Outcome Payout ($) Probability Weighted Value
No accident $0 0.90 0 × 0.90 = $0
Minor accident $2,000 0.08 2,000 × 0.08 = $160
Major accident $15,000 0.018 15,000 × 0.018 = $270
Total loss $50,000 0.002 50,000 × 0.002 = $100
E[payout] — Expected cost per customer per year = $530
E[X] = 0(0.90) + 2000(0.08) + 15000(0.018) + 50000(0.002)
      = 0 + 160 + 270 + 100
      = $530 per customer per year
⚠️ The insurer must charge more than $530 to be profitable. The mean only helps AFTER collecting real claim data — expected value lets them plan BEFORE.
What if we just used the Mean?
1
Last year, 1000 customers filed claims: 80 minor ($2,000), 18 major ($15,000), 2 total loss ($50,000), 900 none.
2
Total paid = 80×2000 + 18×15000 + 2×50000 = $160,000 + $270,000 + $100,000 = $530,000
3
Mean per customer = $530,000 / 1000 = $530 — exactly the same! ✅
✨ Law of Large Numbers: With 1000 customers, the mean from real data ≈ expected value from probabilities. That's why the formula works!

Section 05 — Applications
When to Use Each in Data Science
Knowing which tool to reach for makes all the difference in real projects.
Aspect Mean (x̄) Expected Value E[X]
Input data type Observed / collected data Probabilities + outcomes
When to use After data collection (EDA, reporting) Before data (decision making, risk)
Time orientation Describes the past Predicts the future
Formula x̄ = Σxᵢ / n E[X] = Σ xᵢ·P(xᵢ)
Example in DS Average model accuracy across 5 folds Expected revenue from a recommendation
Used in EDA Statistics Feature engineering ML loss functions Reinforcement learning Risk modeling
Sensitive to outliers Yes — a single outlier shifts mean Yes — rare high-cost events matter!
Relationship With infinite data → Mean converges to Expected Value (Law of Large Numbers)
🤖
Machine Learning
Loss functions like MSE use expected value of errors over the data distribution. Cross-validation computes the mean accuracy.
🎮
Reinforcement Learning
Agents maximize expected reward E[R]. After many episodes, the mean reward is used to evaluate the policy.
📈
A/B Testing
Compute the mean conversion rate for each variant. Use expected value to decide which variant to ship (accounting for revenue).
💳
Credit Risk
Banks compute expected loss = P(default) × Loss Given Default. Historical default rates give the mean to estimate probabilities.
🛍️
Recommendation Systems
Predict expected revenue from each recommendation. Train models on mean historical purchase values per user segment.
🧬
Statistics & Inference
The mean is the sample statistic. The expected value is the population parameter μ that the sample mean estimates.
Section 06 — Summary
Key Takeaways
1
Mean is for summarizing data you already have. Add up all values, divide by count. Simple, powerful, descriptive.
2
Expected Value is for predicting the average outcome of a random process using known probabilities. Essential for decisions under uncertainty.
3
Law of Large Numbers: As sample size → ∞, the sample mean converges to the expected value. This is why both concepts are deeply linked.
4
In data science, you use both. Mean for EDA and reporting; Expected Value for modeling, risk, and decision making. Master both!
🎓 Final Rule of Thumb: If you're looking at data you collected → use the Mean. If you're modeling what will happen using probabilities → use the Expected Value.
