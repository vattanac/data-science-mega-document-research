Machine Learning Model Evaluation

Bias-Variance Tradeoff

Understanding the fundamental tradeoff between model simplicity and complexity

What Is Bias? What Is Variance?

Bias

Bias refers to the error introduced by approximating a real-world problem with a simplified model. When a model has high bias, it makes strong assumptions about the data and fails to capture the underlying patterns.

Key Insight: High bias leads to underfitting, where the model is too simple and performs poorly on both training and test data. Example: fitting a straight line to data that follows a nonlinear curve.

Variance

Variance refers to the sensitivity of the model to fluctuations in the training data. When a model has high variance, it overreacts to individual data points and noise in the training set.

Key Insight: High variance leads to overfitting, where the model fits the training data too closely, including noise, and generalizes poorly to new data. Example: a 10th-degree polynomial fitted to a handful of points.

Underfitting (High Bias)

Symptoms:

  • Poor training accuracy
  • Poor test accuracy
  • Model is too simple
  • Misses key patterns

Causes:

  • Insufficient model capacity
  • Too few features
  • Strong regularization

Overfitting (High Variance)

Symptoms:

  • Excellent training accuracy
  • Poor test accuracy
  • Model is too complex
  • Memorizes noise

Causes:

  • Excessive model capacity
  • Too many features
  • Insufficient training data

The Bias-Variance Tradeoff

The fundamental idea is that as you increase model complexity:

  • Bias decreases — the model becomes more flexible and can better capture complex patterns
  • Variance increases — the model becomes more sensitive to training data variations

There exists an optimal complexity that minimizes total error. Finding this sweet spot is the central challenge in machine learning.

Total Error = Bias² + Variance + Irreducible Error where: • Bias² = (E[ŷ] - y)² • Variance = E[(ŷ - E[ŷ])²] • Irreducible Error = noise inherent in the problem
Irreducible Error: Also called noise, this is the minimum error that cannot be reduced regardless of model choice. It represents the variance in the target variable that cannot be explained by the features.

As model complexity increases, we move along a curve: high bias (left) → optimal (middle) → high variance (right). The goal is to find the sweet spot that minimizes total test error.

Regularization: Controlling Variance

Regularization techniques add constraints to the learning process to reduce variance and prevent overfitting:

L2 Regularization (Ridge)

Adds a penalty proportional to the square of feature coefficients.

Cost = MSE(y, ŷ) + λ ∑(w²) λ (lambda) = regularization strength

Effects:

  • Shrinks large coefficients
  • Keeps all features
  • Smooth decision boundaries
  • Reduces variance

L1 Regularization (Lasso)

Adds a penalty proportional to the absolute value of feature coefficients.

Cost = MSE(y, ŷ) + λ ∑(|w|) λ (lambda) = regularization strength

Effects:

  • Can shrink to zero
  • Feature selection
  • Sparse models
  • Interpretability
Strategy: Start with no regularization, observe overfitting on validation data, then gradually increase λ until validation performance stabilizes. This trades some training accuracy for better generalization.

Practical Tips for Managing Bias-Variance

Diagnosing the Problem

  1. Compare training vs. validation error
    • Both high? Likely high bias (underfitting)
    • Training low, validation high? Likely high variance (overfitting)
  2. Plot learning curves showing error vs. training set size. Converging curves = bias problem. Diverging curves = variance problem.
  3. Use cross-validation to get robust estimates of generalization error.

Solutions

  1. For High Bias (Underfitting)
    • Increase model complexity
    • Add more relevant features
    • Decrease regularization strength
    • Train longer (for neural nets)
  2. For High Variance (Overfitting)
    • Increase training data
    • Reduce model complexity
    • Increase regularization strength
    • Use ensemble methods
    • Early stopping (for neural nets)

Cross-Validation

K-fold cross-validation splits data into K subsets, training K models and averaging results. This provides a robust estimate of generalization error and reduces variance in your performance estimates.

K-Fold Process: 1. Split data into K equal folds 2. For each fold i: - Use fold i as validation - Train on remaining K-1 folds - Record validation error 3. Average the K validation errors

Ensemble Methods

Combining multiple models reduces variance without significantly increasing bias.

Bagging

Train models on random subsets, average predictions. Reduces variance.

Boosting

Sequential training focusing on hard examples. Reduces bias and variance.

Stacking

Meta-learner combines base models. Reduces overall error.

Applications in Machine Learning

Model Selection

When choosing between models, consider their bias-variance characteristics:

  • Linear models (linear regression, logistic regression): High bias, low variance. Good starting point, especially with small datasets.
  • Tree-based models (decision trees, random forests): Flexible bias-variance tradeoff. Trees have high variance but forests reduce it.
  • Neural networks: Can represent any function (low bias) but require regularization and large datasets to prevent overfitting (reduce variance).
  • Support vector machines: Configurable via kernel choice and regularization parameter C.

Hyperparameter Tuning

Different hyperparameters control the bias-variance tradeoff:

Tree Depth

↑ depth → ↓ bias, ↑ variance

Learning Rate

↑ rate → ↓ training time, ↑ noise

Regularization λ

↑ λ → ↑ bias, ↓ variance

Data Collection

More data always helps:

  • Reduces variance: Larger datasets make model predictions more stable across different training runs.
  • Allows complex models: With more data, you can safely use higher-capacity models without overfitting.
  • Improves all metrics: More diverse data helps both bias and variance.
